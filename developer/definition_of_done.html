Definition of Done

Tip

For our 2-3 Developer Team: Efficiency is key. We rely on Automated Checks to be the primary gatekeepers. If the build fails, the tests fail, or the analyzer warnings persist, the task is not done.

The Manual Checklist below is for the "Human Element"—verification of logic, design correctness, and user experience that simple scripts cannot catch.

1. Automated Checks (The Gatekeepers)

Before you even open the PR, ensure these are Green.

  • [ ] Build & CI
    • [ ] Solution builds with Zero Warnings.
    • [ ] All Tests Passing (Unit, Integration, Architecture/ArchUnitNET).
    • [ ] 80% Code Coverage on Core Layers.
  • [ ] Code Quality
    • [ ] EditorConfig rules pass (naming, formatting).
    • [ ] PR Title follows Conventional Commits (e.g., feat: add user notification).

2. Design Correctness (Decision Prompts)

Verify using the prompts in Patterns and Principles:

  • [ ] BDD: Did I write the Gherkin scenario before the code?
  • [ ] DDD: Does each piece of code live in the correct layer? (Use the decision tree)
  • [ ] CQS: Commands mutate, Queries read. No mixed concerns.

3. Implementation Correctness (Manual Checklist)

Handlers & Logic

  • [ ] Attributes Correctly Applied:
    • [ ] [RequireEntitlement(EntitlementType.WordStandardTools)] (if feature requires license).
    • [ ] [DocumentRequired] (if handler operates on an active document).
  • [ ] Return Values:
    • [ ] Return Result.Success() or Result<T> values.
    • [ ] Do NOT return null or throw exceptions for control flow.
    • [ ] If operation cannot continue, return Result.Failure.
  • [ ] User Feedback:
    • [ ] Use IUserNotifier for any user-facing messages.
    • [ ] Must be Localized (no hardcoded strings in logic).

COM Interop

  • [ ] Resource Management:
    • [ ] All COM objects released explicitly (using using blocks or Marshal.ReleaseComObject in finally).
    • [ ] Two-dot rule followed (avoid obj.Prop.Prop chains).
  • [ ] Layer Safety:
    • [ ] No COM types leaked into Core/Domain layers.

Configuration

  • [ ] Settings:
    • [ ] [Description] attribute added to all new settings props.
    • [ ] Validation logic implemented (e.g., check paths exist at load time).
  • [ ] Sample Configs:
    • [ ] Debug/Sample configurations added to: C:\Brochet Dropbox\Brochet internal\Development\Word\Config

Documentation

  • [ ] Developer Docs: Updated docs/developer with "Why" not just "What".
  • [ ] User Docs: Updated docs/user if feature is visible.
  • [ ] Schemas: Validated (if changing config structure).
Note

Why this matters: As a small team, we don't have a dedicated QA department. This Definition of Done ensures that when we merge, we are confident the solution is stable, maintainable, and ready for deployment.
