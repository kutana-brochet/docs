Dependency Injection for the forgetful

Adding a command

DependencyInjection.cs registers all Handlers that use ICommandHandler So when you run:

var command = new AttachTemplateOnOpen.Command();
CommandDispatcher.Dispatch<AttachTemplateOnOpen.Command, Result>(command, CancellationToken.None);

This is registered in DI and the handler is run

Adding implementation

The command handler may need to talk directly to the implementation, such as sending a message to the user Add the following to register the implementation with DI

.AddSingleton<IUserNotifier, UserNotifier>()

Domain Events

Domain events are used to decouple side effects from command handling, allowing event handlers to respond to changes in application state independently of command execution.

.AddSingleton<IDomainEventHandler<ToggleQuotesEvent>, ToggleQuotesEventhandler>()

Registering DocumentEvents

DocumentEvents must be registered with DI to manage its lifetime and ensure that event handlers are properly attached and detached from Word documents.

.AddSingleton<DocumentEvents>()
