Square Brackets - Developer Guide

Technical implementation guide for the Square Brackets feature. This feature provides tools for working with bracket pairs [...] in legal documents through navigation, addition, and deletion commands.

Architecture Overview

The Square Brackets feature follows Clean Architecture patterns with clear separation of concerns:

WordAddIn (Presentation)
  └─> ProductRibbon.cs - Ribbon button handlers
  
Core.Application (Use Cases)
  └─> Document/SquareBrackets/
      ├─> FindSquareBrackets.cs - Navigation command
      ├─> AddSquareBrackets.cs - Add brackets command
      ├─> DeleteSquareBrackets.cs - Delete commands
      ├─> InsertBlob.cs - Insert blob command
      └─> Blob.cs - Blob value object

Core.Domain (Entities)
  └─> Documents/
      └─> Delimiter.cs - Delimiter value object

Infrastructure.Persistence (Data)
  └─> Word/
      ├─> SquareBracketsConfiguration.cs - Config model
      └─> SquareBracketsSettingsRepository.cs - Config access

Key Architectural Principles

  1. CQRS Pattern: All operations are commands dispatched through ICommandDispatcher
  2. Strategy Pattern: Search and modification behaviors use pluggable strategy objects
  3. Value Objects: Delimiter and Blob are immutable value objects with factory methods
  4. Railway-Oriented Programming: All commands return Result<TValue, Error>
  5. Clean Architecture: Domain has no infrastructure dependencies; COM interop isolated in VSTO layer

Layer Structure

Domain Layer (Core.Domain.Documents)

Delimiter

Immutable value object representing opening and closing delimiter characters.

public record Delimiter(string Open, string Close)
{
    public static Delimiter Default() => new("[", "]");
}

Application Layer (Core.Application)

FindSquareBrackets Command

Locates and selects square bracket pairs using the Strategy pattern for directional search.

Command Definition:

public class Command(bool searchForward) : ICommand
{
    public bool SearchForward { get; } = searchForward;
}

Handler Flow:

  1. Get current selection from IDocumentEditor
  2. Select appropriate search strategy (ForwardSearchStrategy or BackwardSearchStrategy)
  3. Execute search via IDelimiterPairFinder
  4. If found: select the bracket pair
  5. If not found: notify user or prompt to restart from beginning/end

Search Strategies:

  • ForwardSearchStrategy - Searches from cursor position toward end of document
  • BackwardSearchStrategy - Searches from cursor position toward start of document

Returns:

  • Result.Success<Unit, Error> - Bracket pair found and selected
  • Result.Success<Unit, Error> - No brackets exist (user notified)
  • Result.Success<Unit, Error> - End of document reached, restart prompt shown
  • Result.Failure<Unit, Error> - Infrastructure error occurred

Special Handling:

  • Placeholder content controls showing brackets are treated as bracket pairs
  • Allows consistent navigation across both text brackets and content control placeholders

AddSquareBrackets Command

Wraps the current selection in square brackets using AddDelimiterPairStrategy.

Command Definition:

public class Command(Delimiter delimiter) : ICommand
{
    public Delimiter Delimiter { get; } = delimiter;
    
    public static Command Default => new(Delimiter.Default());
}

Handler Flow:

  1. Get delimiter configuration from ISquareBracketsSettingsRepository
  2. Execute AddDelimiterPairStrategy via IDocumentModifier
  3. Expand selection to include new brackets

Behaviour:

  • Inserts opening delimiter before selection start
  • Inserts closing delimiter after selection end
  • Expands selection to include new brackets
  • Works with empty selection (inserts [] at cursor)
  • Creates nested brackets if text already contains brackets

Returns:

  • Result.Success<Unit, Error> - Brackets added successfully
  • Result.Failure<Unit, Error> - Document modification failed

Entitlements: WordStandardTools

Decorators: [DocumentRequired]

DeleteSquareBrackets Command

Removes brackets and/or content using configurable deletion strategies.

Command Definition:

public class Command(DeletionMode mode, Delimiter delimiter) : ICommand
{
    public DeletionMode Mode { get; } = mode;
    public Delimiter Delimiter { get; } = delimiter;
    
    public static Command DeleteDelimiterPair => new(DeletionMode.DeleteDelimiterPair, Delimiter.Default());
    public static Command DeleteAll => new(DeletionMode.DeleteAll, Delimiter.Default());
}

Handler Flow:

  1. Validate selection starts with [ and ends with ]
  2. If invalid: notify user and return success (no-op)
  3. Select appropriate deletion strategy based on mode
  4. Execute strategy via IDocumentModifier

Deletion Strategies:

  • DeleteDelimiterPair - Removes only [ and ], keeps content
  • DeleteContent - Removes content, keeps []
  • DeleteAll - Removes everything including brackets

Validation:

  • Selection must start with opening delimiter
  • Selection must end with closing delimiter
  • Invalid selections show user notification, document unchanged

Returns:

  • Result.Success<Unit, Error> - Deletion completed
  • Result.Success<Unit, Error> - Invalid selection (user notified)
  • Result.Failure<Unit, Error> - Document modification failed

Entitlements: WordStandardTools

Decorators: [DocumentRequired]

InsertBlob Command

Replaces the current selection with a formatted marker between delimiters.

Command Definition:

public class Command(Delimiter delimiter, Blob blob) : ICommand
{
    public Delimiter Delimiter { get; } = delimiter;
    public Blob Blob { get; } = blob;
    
    public static Command Default => new(Delimiter.Default(), Blob.Default());
}

Handler Flow:

  1. Read blob configuration from ISquareBracketsSettingsRepository
  2. Build replacement text: [{blob.Text}]
  3. Replace selection via IDocumentModifier.ReplaceText()
  4. Apply font formatting to blob text only (not delimiters)

Behaviour:

  1. Reads blob configuration from SquareBracketsConfiguration
  2. Replaces selection with [{blob text}]
  3. Applies font formatting to blob text only (not delimiters)
  4. Uses Wingdings font when blob text is empty

Returns:

  • Result.Success<Unit, Error> - Blob inserted successfully
  • Result.Failure<Unit, Error> - Document modification failed

Entitlements: WordStandardTools

Decorators: [DocumentRequired]

Blob Value Object

Immutable value object representing blob insertion configuration.

public record Blob(string Text, string? FontName, double? FontSize)
{
    public static Blob Default() => new(string.Empty, "Wingdings", 10);
}

Infrastructure Layer (Infrastructure.Persistence.Word)

SquareBracketsConfiguration

Configuration model for Square Brackets operations.

public class SquareBracketsConfiguration
{
    public Delimiter Delimiter { get; set; } = Delimiter.Default;
    public Blob? Blob { get; set; }
}

SquareBracketsSettingsRepository

Implements ISquareBracketsSettingsRepository to load configuration from JSON.

public interface ISquareBracketsSettingsRepository
{
    Delimiter GetDelimiter();
    Maybe<Blob> GetBlob();
}

Implementation reads from SquareBracketsConfiguration section with fallback to defaults:

  • GetDelimiter() returns Delimiter.Default() if not configured
  • GetBlob() returns Maybe.None if not configured, allowing caller to use Blob.Default()

Strategy Pattern Implementation

Square Brackets commands use the Strategy pattern to support multiple operational modes.

Search Strategies

Interface:

public interface ISearchStrategy
{
    Maybe<Range> Execute(Range startRange, Delimiter delimiter);
}

Implementations:

  • ForwardSearchStrategy - Searches from cursor toward document end
  • BackwardSearchStrategy - Searches from cursor toward document start

Both strategies use Word's Range.Find API for optimal performance.

Modification Strategies

Interface:

public interface IDelimiterPairStrategy
{
    Result<Unit, Error> Execute(Range selection, Delimiter delimiter);
}

Implementations:

  • AddDelimiterPairStrategy - Inserts delimiters around selection
  • RemoveDelimiterPairStrategy - Removes delimiters, preserves content
  • RemoveContentStrategy - Removes content, preserves delimiters

See Strategy Pattern Guide for detailed implementation patterns.

Extension Points

Custom Delimiters

Override default delimiters via configuration to support different bracket types:

{
  "SquareBracketsConfiguration": {
    "Delimiter": {
      "Open": "{",
      "Close": "}"
    }
  }
}

Important: Custom delimiters affect all Square Brackets operations. Ensure your choice doesn't conflict with other document syntax (e.g., mail merge fields use {{ }}).

Custom Blob Markers

Configure blob appearance for organizational standards:

{
  "SquareBracketsConfiguration": {
    "Blob": {
      "Text": "●",
      "FontName": "Arial",
      "FontSize": 12
    }
  }
}

Common patterns:

  • Wingdings checkbox: Text: empty string, FontName: "Wingdings"
  • X marker: Text: "X", FontName: "Arial"
  • Bullet: Text: "●", FontName: "Arial"

Ribbon Visibility Rules

Control when Square Brackets buttons appear using Ribbon Item Rules:

<button id="AddSquareBrackets"

        getVisible="GetVisibleById"
        getEnabled="GetEnabledById"
        label="Add Brackets" />

Configuration:

      }

Performance Considerations

COM Object Disposal

All Word interop objects are released in finally blocks to prevent memory leaks:

Word.Range range = null;
try
{
    range = document.Range();
    // Operations
}
finally
{
    if (range != null) Marshal.ReleaseComObject(range);
}

This pattern appears 73 times in the codebase and is critical for VSTO stability.

Error Handling

All commands return Result<TValue, Error> following railway-oriented programming:

var result = _commandDispatcher.Dispatch<FindSquareBrackets.Command, Unit>(command);
if (result.IsFailure)
{
    _userNotifier.Notify(result.Error);
}

Common Error Codes

Errors are returned from the AddDelimiterPairStrategy and propagated through the handler.

Infrastructure errors:

Error Code Meaning
Document.ModificationFailed Word interop error
Document.NoActiveDocument No document open

Localisation

User-facing messages are localised via ILocalisationService and DomainMessages.resx.

Adding Translations:

Create DomainMessages.{locale}.resx files:

  • DomainMessages.fr-FR.resx - French
  • DomainMessages.de-DE.resx - German
  • etc.

Messages are resolved automatically based on Thread.CurrentThread.CurrentUICulture.

COM Safety Patterns

Document Validation

Always check for active document before operations:

if (Application.Documents.Count == 0)
{
    return Result.Failure<Unit, Error>(DomainErrors.Document.NoActiveDocument);
}

Exception Handling

Catch COM-specific exceptions:

try
{
    // Word operations
}
catch (System.Runtime.InteropServices.COMException ex)
{
    _logger.LogError(ex, "COM error in Square Brackets operation");
    return Result.Failure<Unit, Error>(DomainErrors.Document.ModificationFailed);
}

Safe Word API Calls

Validate COM objects before use:

if (selection is null || !Marshal.IsComObject(selection))
{
    return Result.Failure<Unit, Error>(DomainErrors.Document.InvalidSelection);
}
