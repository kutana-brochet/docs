Ribbon Item Rules - Developer Guide

Technical implementation guide for the rules-based system that controls ribbon item visibility and enabled state. This system runs in parallel with the existing script-based visibility (GetVisibleByScript/GetEnabledByScript) but targets non-script ribbon elements such as dynamic menus and groups.

Architecture Overview

Ribbon invalidated (startup, config change, document event)
  |
  v
Office calls getVisible/getEnabled callback on each ribbon element
  |
  v
ProductRibbon.GetVisibleById(control) / GetEnabledById(control)
  |
  v
GetVisible.Query(control.Id) / GetEnabled.Query(control.Id)
  |
  v
ItemRuleRepository.GetIsVisible(id) / GetIsEnabled(id)
  |-- Iterates RulesConfiguration.RibbonItemRules in order
  |-- First glob pattern match wins
  |-- "Always" or empty → true
  |-- Otherwise → delegates to IApplicationRulesEngine.IsVisible/IsEnabled(workflowName)
  |
  v
Maybe<bool> → Handler applies default (visible=true, enabled=false)
  |
  v
bool returned to Office

Design Decisions

  1. Defaults: Visible defaults to true (items don't vanish), Enabled defaults to false (items are greyed-out until explicitly enabled). This prevents users from clicking unconfigured items.
  2. Glob patterns: Uses DotNet.Glob for pattern matching. Chosen over regex for simplicity and familiarity.
  3. First-match-wins: Rules are evaluated in declaration order. This allows specific overrides before catch-all patterns.
  4. Maybe return: The repository returns Maybe<bool>.None when no rule matches, allowing each handler to apply its own default.

Layer Structure

Application Layer (Core.Application.Rules)

GetVisible.Query / GetEnabled.Query

Thin query handlers that delegate to IItemRuleRepository:

public static class GetVisible
{
    public record Query(string Id) : IQuery;

    public class Handler(IItemRuleRepository repository) : IQueryHandler<Query, bool>
    {
        public Result<bool, Error> Handle(Query query)
        {
            var maybeVisible = repository.GetIsVisible(query.Id);
            return maybeVisible.GetValueOrDefault(true);  // default: visible
        }
    }
}

public static class GetEnabled
{
    public record Query(string Id) : IQuery;

    public class Handler(IItemRuleRepository repository) : IQueryHandler<Query, bool>
    {
        public Result<bool, Error> Handle(Query query)
        {
            var maybeEnabled = repository.GetIsEnabled(query.Id);
            return maybeEnabled.GetValueOrDefault();  // default: false (disabled)
        }
    }
}

RuleConstants

public static class RuleConstants
{
    public const string Always = "Always";

    public static bool IsAlways(string value) =>
        string.IsNullOrWhiteSpace(value) || value.Equals(Always, StringComparison.OrdinalIgnoreCase);
}

"Always", empty string, and null all resolve to unconditionally true — simplifies configuration.

IItemRuleRepository

public interface IItemRuleRepository
{
    Maybe<bool> GetIsVisible(string id);
    Maybe<bool> GetIsEnabled(string id);
}

Returns Maybe<bool>.None when no rule matches, allowing the handler to apply its domain-specific default.

Infrastructure Layer (Infrastructure.Persistence.Rules)

RulesConfiguration

public class RulesConfiguration
{
    public IEnumerable<Workflow> Workflows { get; init; } = [];
    public IList<ItemMappingDto> RibbonItemRules { get; init; } = [];
}

Bound from RulesConfiguration JSON section via IOptionsMonitor<RulesConfiguration>. Hot-reloads on file change.

ItemMappingDto

public record ItemMappingDto
{
    [Required]
    public string Pattern { get; init; } = string.Empty;

    public string EnabledWhen { get; init; } = RuleConstants.Always;
    public string VisibleWhen { get; init; } = RuleConstants.Always;
}

ItemRuleRepository

Core matching logic:

public class ItemRuleRepository(
    IOptionsMonitor<RulesConfiguration> ribbonItemRulesConfig,
    IApplicationRulesEngine rulesEngine) : IItemRuleRepository
{
    private readonly ConcurrentDictionary<string, Glob> _globCache = new();

    private Maybe<bool> Evaluate(
        string id,
        Func<ItemMappingDto, string> ruleSelector,
        Func<string, Maybe<bool>> engineEvaluator)
    {
        foreach (var rule in ribbonItemRulesConfig.CurrentValue.RibbonItemRules)
        {
            if (string.IsNullOrWhiteSpace(rule.Pattern))
                continue;

            if (rule.Pattern != "*" && !GetOrParseGlob(rule.Pattern).IsMatch(id))
                continue;

            var whenValue = ruleSelector(rule);
            return RuleConstants.IsAlways(whenValue) ? true : engineEvaluator(whenValue);
        }

        return Maybe<bool>.None;
    }
}

Key behaviours:

  • Glob cache: Parsed glob patterns are cached in a ConcurrentDictionary for performance — ribbon callbacks fire frequently during invalidation
  • Wildcard shortcut: "*" pattern bypasses glob parsing (always matches)
  • Empty pattern skip: Rules with blank patterns are silently ignored
  • First match wins: Iteration stops at the first matching pattern

VSTO Layer (WordAddIn.Ribbon)

ProductRibbon Callbacks

Four callbacks are available for use in Ribbon XML:

public bool GetVisibleById(IRibbonControl control) => GetVisible(control.Id);
public bool GetEnabledById(IRibbonControl control) => GetEnabled(control.Id);
public bool GetVisibleByTag(IRibbonControl control) => GetVisible(control.Tag);
public bool GetEnabledByTag(IRibbonControl control) => GetEnabled(control.Tag);

The ById variants match against the element's id attribute; the ByTag variants match against tag. Both delegate to the same query pipeline.

Private helpers apply safe defaults when the value is null/empty or when the query fails:

private bool GetVisible(string value)
{
    if (string.IsNullOrEmpty(value)) return true;
    var query = new GetVisible.Query(value);
    var result = _queryDispatcher.Dispatch<GetVisible.Query, bool>(query);
    return result.GetValueOrDefault(defaultValue: true);
}

private bool GetEnabled(string value)
{
    if (string.IsNullOrEmpty(value)) return false;
    var query = new GetEnabled.Query(value);
    var result = _queryDispatcher.Dispatch<GetEnabled.Query, bool>(query);
    return result.GetValueOrDefault(defaultValue: false);
}

Ribbon Refresh

RulesConfiguration is registered with IOptionsMonitor. On configuration file change:

options.OnChange(_ =>
{
    Globals.ThisAddIn.UiSyncContext.Post(_ => Refresh(), null);
});

Refresh() calls RibbonUi.Invalidate(), which triggers Office to re-query all getVisible/getEnabled callbacks.

Configuration Format

{
  "RulesConfiguration": {
    "Workflows": [
      {
        "WorkflowName": "RibbonRules",
        "Rules": [
          {
            "RuleName": "DocumentIsOpen",
            "Expression": "ActiveDocumentName != null"
          }
        ]
      }
    ],
    "RibbonItemRules": [
      {
        "Pattern": "newDocumentMenu",
        "VisibleWhen": "Always",
        "EnabledWhen": "DocumentIsOpen"
      },
      {
        "Pattern": "starter*",
        "VisibleWhen": "Always",
        "EnabledWhen": "DocumentIsOpen"
      },
      {
        "Pattern": "*",
        "VisibleWhen": "Always",
        "EnabledWhen": "Always"
      }
    ]
  }
}

Relationship to Script Visibility

The codebase has two parallel visibility/enabled systems:

Script System Ribbon Item Rules
Ribbon callbacks GetVisibleByScript / GetEnabledByScript GetVisibleById / GetEnabledById / ByTag
Configuration ScriptConfiguration.Mappings[id].VisibleWhen RulesConfiguration.RibbonItemRules[].VisibleWhen
Matching Exact id lookup in dictionary Glob pattern matching (first match)
Repository ScriptRepository ItemRuleRepository
Target elements Buttons with onAction="RunScript" Any ribbon element (menus, groups, buttons)
Rules engine Same IApplicationRulesEngine Same IApplicationRulesEngine

Both systems resolve workflow names through IApplicationRulesEngine, so the same workflow definitions work for both.

Adding a New Callback Variant

To add a new matching strategy (e.g. matching by a custom ribbon attribute):

  1. Add a new public method in ProductRibbon.cs that extracts the value from IRibbonControl
  2. Delegate to the existing GetVisible(string) / GetEnabled(string) private methods
  3. Reference the new callback name in Ribbon XML

No changes needed in the Application or Infrastructure layers.
