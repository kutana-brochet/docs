Document Menus - Developer Guide

Technical implementation guide for the New Document, Starter Document, and Insert Snippet dynamic menu subsystem. All three features share a common discovery, formatting, and rendering pipeline but differ in their final action.

Menu Type Action Creates New Document? Default Extensions
New Document CreateDocument Yes (from template) .dotx, .dotm
Starter Document CreateFromStarterDocument Yes (from existing doc) .docx, .docm, .dotx, .dotm
Insert Snippet InsertSnippet No (inserts into active doc) .docx, .docm

Architecture Overview

The document menu subsystem implements a pipeline pattern: file discovery, menu formatting, XML rendering, and command dispatch.

Ribbon click (dynamicMenu getContent callback)
  |
  v
GetDocumentMenuItems.Query --> FileProviderRepository --> FileDiscoveryService
  (reads config)                (scans file system)
  |
  v
FormatDocumentMenu.Query --> FolderTreeBuilder + DocumentMenuFormatter
  |
  v
DocumentMenuXmlRenderer --> XML string returned to Office
  |
  v
User picks item --> Ribbon onAction callback
  |
  v
RibbonTagCache.TryGet(tag) --> full file path
  |
  v
CreateDocument / CreateDocumentFromStarterDocument / InsertSnippet command

Key Architectural Principles

  1. Shared Pipeline: All three menu types use the same discovery, formatting, and rendering infrastructure
  2. Menu-Type Discrimination: DocumentMenuType enum distinguishes behaviour at each pipeline stage
  3. Configuration-Driven: Menu content is entirely driven by FileProviderConfiguration JSON
  4. Tag-Based Security: File paths are never exposed in ribbon XML; RibbonTagCache maps opaque tags to paths
  5. Clean Architecture: Domain has no infrastructure dependencies; COM interop is isolated in the VSTO layer

Layer Structure

Domain Layer (Core.Domain.Word)

IDocumentSource

Base abstraction for any file that can appear in a menu:

public interface IDocumentSource
{
    string Name { get; }       // Filename (e.g. "Contract.dotx")
    string Location { get; }   // Full path
    string Container { get; }  // Parent folder path
    string Extension { get; }
    bool Matches(string value);
    bool MatchesContainer(IDocumentSource source);
}

DocumentFile

Sealed record implementing IDocumentSource for files on disk. Created via factory method:

public static Maybe<DocumentFile> Create(string path, bool exists)

Returns Maybe.None if the file does not exist.

DocumentPath

Validated path value object:

public sealed record DocumentPath
{
    public static Result<DocumentPath, Error> Create(string path)
}

Rejects null/whitespace paths with DomainErrors.Document.InvalidPath.

Template

Entity representing a Word template (.dotx/.dotm). Extends TemplateReference:

public record Template(IDocumentSource Source, TemplateMetadata Metadata)

Key behaviours:

  • MatchesFilter(string) - regex match against file name (used for pre-profile configuration)
  • IsMissing() - true if Source is a MissingDocument

StarterDocument

Pairs a source file with its resolved template:

public record StarterDocument(IDocumentSource Source, Template Template)

The Template is resolved by inspecting the starter file's XML to read its AttachedTemplate property.

Application Layer (Core.Application)

DocumentMenuType

public enum DocumentMenuType { NewDocument, StarterDocument, Snippet }

MenuAction

Immutable record mapping menu types to ribbon callback names:

public static readonly MenuAction CreateDocument            = new("CreateDocument", "FileNewBlankDocument");
public static readonly MenuAction CreateFromStarterDocument = new("CreateFromStarterDocument", "FileNewBlankDocument");
public static readonly MenuAction InsertSnippet             = new("InsertSnippet", "FileNewBlankDocument");

The ActionName is written into the generated ribbon XML as the onAction attribute value.

GetDocumentMenuItems.Query

Retrieves raw file list for a given menu button ID:

public class Query(DocumentMenuType menuType, string id) : IQuery

Returns IReadOnlyList from IFileProviderRepository.GetDocuments().

FormatDocumentMenu.Query

Transforms raw documents into a structured DocumentMenu:

public class Query(DocumentMenuType menuType, IReadOnlyList<IDocumentSource> documents) : IQuery

Delegates to IDocumentMenuFormatter which uses FolderTreeBuilder to group items.

DocumentMenu / DocumentMenuGroup / DocumentMenuItem

public record DocumentMenu(IReadOnlyList<DocumentMenuGroup> Groups, bool UseSubmenus, Maybe<string> Title);
public record DocumentMenuGroup(string DisplayName, IReadOnlyList<DocumentMenuItem> Items, ...);
public record DocumentMenuItem(string DisplayName, IDocumentSource Source, MenuAction Action);

Commands

CreateDocument.Command (DocumentManagement/CreateDocument.cs):

public class Command : ICommand
{
    public string Value { get; }       // Template name or path
    public string DisplayName { get; }
}

Handler flow:

  1. ITemplateRepository.GetByNameOrPath() - resolve template
  2. IDocumentManagementService.CreateDocument(template) - Word Documents.Add()
  3. Optional pre-profile via DMS configuration
  4. Dispatch DocumentCreatedFromTemplateEvent

Entitlements: WordDocumentManagement + WordStandardTools

CreateDocumentFromStarterDocument.Command (DocumentManagement/CreateDocumentFromStarterDocument.cs):

public class Command(string starterDocumentPath) : ICommand

Handler flow:

  1. DocumentFileFactory.Create(path) - validate path + existence
  2. IDocumentRepository.GetStarterDocument(documentFile) - inspect XML, resolve template
  3. IDocumentManagementService.CreateDocument(starterDocument) - Word Documents.Add()
  4. Optional pre-profile via DMS configuration
  5. Dispatch DocumentCreatedFromStarterDocument event

Entitlements: WordDocumentManagement + WordStandardTools

InsertSnippet.Command (Word/InsertSnippet.cs):

public class Command(string sourcePath) : ICommand

Handler flow:

  1. DocumentPath.Create(sourcePath) - validate path
  2. IDocumentEditor.InsertFile(documentPath) - Word Selection.InsertFile()

Entitlements: WordStandardTools. Decorators: [DocumentRequired]

Application Events

  • DocumentCreatedFromTemplateEvent - raised after CreateDocument succeeds
  • DocumentCreatedFromStarterDocument - raised after CreateDocumentFromStarterDocument succeeds

Both implement IApplicationEvent and are dispatched asynchronously for side effects (DMS profiling, VBA callbacks).

NB: Insert Snippet does not raise an application event.

Infrastructure Layer

FileProviderRepository (Infrastructure.Persistence)

Implements IFileProviderRepository. Reads FileProviderConfiguration and delegates to FileDiscoveryService:

public IReadOnlyList<IDocumentSource> GetDocuments(DocumentMenuType menuType, string id)

Resolves path placeholders (%Workgroup%, %User%) via IEnvironmentService.GetPaths().

FileDiscoveryService

Recursively walks the filesystem up to MaxDepth levels. Exclusions:

  • Temporary lock files (~$ prefix)
  • Non-matching extensions
  • Non-matching prefixes (when configured)
  • Directories in ExcludedContainers
  • Reparse points (symlinks / junctions)

FolderTreeBuilder

Builds the group hierarchy from IDocumentSource lists:

  • Finds common root path, strips it
  • Compacts single-child directory chains: "Parent / Child" instead of nested menus
  • Max depth: 2 levels (deeper items flattened into level 2)
  • Items within a group sorted alphabetically
  • Optional leading-number stripping from names

DisplayNameFormatter

  • Strips file extensions from display names
  • Strips numeric prefixes (01_, 01-, 01., 01 , 01) unless the entire name is digits (e.g. locale code 1033)

VSTO Layer (WordAddIn)

ProductRibbon Callbacks

Content callbacks:

public string GetNewDocuments(IRibbonControl button)
    => GetDocumentMenu(DocumentMenuType.NewDocument, button.Id);

public string GetStarterDocuments(IRibbonControl button)
    => GetDocumentMenu(DocumentMenuType.StarterDocument, button.Id);

public string GetSnippetDocuments(IRibbonControl button)
    => GetDocumentMenu(DocumentMenuType.Snippet, button.Id);

Visibility and enabled callbacks are handled by the Ribbon Item Rules subsystem. Dynamic menus typically use GetVisibleById / GetEnabledById attributes in Ribbon XML:

<dynamicMenu id="newDocumentMenu"
             getVisible="GetVisibleById"
             getEnabled="GetEnabledById"
             getContent="GetNewDocuments" />

Action handlers:

public void CreateDocument(IRibbonControl control)            // dispatches CreateDocument.Command
public void CreateFromStarterDocument(IRibbonControl control) // dispatches CreateDocumentFromStarterDocument.Command
public void InsertSnippet(IRibbonControl control)             // dispatches InsertSnippet.Command

All action handlers resolve the file path from RibbonTagCache using control.Tag.

DocumentMenuXmlRenderer

Renders DocumentMenu to Office Ribbon XML. Per item:

  • Resolves icon via IMenuItemIconResolver
  • Composes screentip with last modified date: "TemplateName (dd/MM/yyyy)"
  • Registers path in RibbonTagCache, embeds tag in tag= attribute
  • For grouped menus: uses menu elements for submenus, menuSeparator for flat separators

MenuItemIconResolver

Resolves icons per menu item. For Starter Documents with staleness configuration, applies colour-coded icons based on file age thresholds.

RibbonTagCache

Thread-safe cache mapping opaque string keys to file paths. Prevents file paths from appearing in generated XML. Each Render() call registers new entries; entries are never removed during the application lifecycle.

Configuration Format

FileProviderConfiguration

Located in the CoreConfiguration JSON file at the shared or user config path.

{
  "FileProviderConfiguration": {
    "NewDocument": { },
    "StarterDocument": { },
    "InsertSnippet": { }
  }
}

Each section is a dictionary keyed by ribbon button id. This allows multiple ribbon buttons to show different file sets from the same menu type.

FileProviderDto

public class FileProviderDto
{
    public LocalSourceDto Source { get; init; }
    public DisplayStyleDto? Style { get; init; }
}

LocalSourceDto

public class LocalSourceDto
{
    public string Location { get; init; } = "%Workgroup%";
    public int? MaxDepth { get; init; } = 0;
    public IReadOnlyList<string> Extensions { get; init; } = [];
    public IReadOnlyList<string> Prefixes { get; init; } = [];
    public IReadOnlyList<string> ExcludedContainers { get; init; } = [];
}

DisplayStyleDto

public class DisplayStyleDto
{
    public string? Title { get; init; }
    public DisplayType Type { get; init; } = DisplayType.Flat;
    public bool RemoveLeadingNumbersFromTemplateNames { get; init; } = true;
    public bool RemoveLeadingNumbersFromContainerNames { get; init; } = true;
    public bool SeparateByContainer { get; init; } = true;
    public StalenessConfigDto? Staleness { get; init; }
}

DisplayType

public enum DisplayType { Flat, Group }
  • Flat: FlatMenuDisplayStyle - all items in one group, optional separator headers by container
  • Group: GroupedMenuDisplayStyle - each container becomes a submenu (max 2 levels)

Adding a New Menu Type

Step 1: Add to DocumentMenuType enum

Add a new value to DocumentMenuType in Core.Application/Document/DocumentMenuType.cs.

Step 2: Create a MenuAction

Add a static MenuAction in Core.Application/Document/MenuActions.cs with the ribbon callback name.

Step 3: Add default extensions

Add default extensions in Core.Application/Document/FileProviderDefaults.cs.

Step 4: Add configuration section

Add a new dictionary property to FileProviderConfiguration in Infrastructure.Persistence/Files/FileProviderConfiguration.cs.

Step 5: Add ribbon callbacks

In WordAddIn/Ribbon/ProductRibbon.cs:

  • Add a GetContent callback that calls GetDocumentMenu() with the new type
  • Add an onAction callback that dispatches the appropriate command

Step 6: Create the command

Create a new command + handler in Core.Application following the existing patterns (CreateDocument.cs or InsertSnippet.cs).

Starter Document Template Resolution

The starter document pipeline has an additional step: template resolution. When a user selects a starter document:

DocumentFileFactory.Create(path)
  |
  v
WordDocumentRepository.GetStarterDocument(documentFile)
  |-- InspectFile(documentSource)                      // Read XML from .docx
  |-- inspection.TemplateName                          // Attached template name/path
  |-- templateRepository.GetByNameOrPath(fullName)     // Try full path
  |-- templateRepository.GetByNameOrPath(fileName)     // Try just filename
  |-- templateRepository.GetDefault()                  // Fallback
  |
  v
StarterDocument(source, resolvedTemplate)

Design Rationale: Starter documents carry their template reference internally rather than via configuration. This means the same starter document always produces a document with the correct template, regardless of which menu or configuration triggered it.

COM Safety

All Word COM operations in the document creation pipeline are wrapped in try/catch with proper COM object disposal. Key patterns:

  • Documents.Add() returns a Document COM object that must be tracked
  • Pre-profile dialogs are synchronous COM calls; if cancelled, the created document may need to be closed
  • Selection.InsertFile() (Insert Snippet) operates on the active selection and does not create new COM objects

COM Interface (External Automation)

External VBA macros or DMS systems can trigger document creation programmatically:

' New Document from template
comInterface.RunOperation("DMSNewDocument", "templateNameOrPath")

' New Document from Starter Document
comInterface.RunOperation("DMSNewDocumentFromStarterDocument", "C:\StarterDocs\Agreement.docx")

Both dispatch the same commands as the ribbon buttons and return the created Word Document COM object.
