Outlook Address Book Configuration

This guide explains how to configure the Outlook address book provider to fetch contacts from Microsoft Outlook.

Overview

The Outlook provider allows you to retrieve contact information from Outlook address books and use it within Word documents for mail merge, data insertion, or other purposes.

Prerequisites

  • Microsoft Outlook must be installed on the system
  • Outlook must be configured with at least one profile
  • The add-in must have permission to access Outlook (check Outlook Trust Center settings)

Configuration

Basic Configuration

Add the following configuration to your dataset configuration file:

{
  "$schema": "../../../output/schemas/DatasetProviderConfiguration.schema.json",
  "DatasetProviderConfiguration": {
    "Datasets": {
      "OutlookContacts": {
        "$connectionType": "Outlook",
        "OutlookAddressbook": "Contacts",
        "ConnectionSettings": "",
        "SQLQuery": "",
        "Timeout": 30,
        "FieldAliasOrder": {
          "FullName": "Name",
          "EmailAddress": "Email",
          "CompanyName": "Company",
          "JobTitle": "Title",
          "BusinessPhone": "Phone",
          "MobilePhone": "Mobile"
        },

Required Properties

$connectionType

  • Value: "Outlook"
  • Description: Specifies that this dataset uses the Outlook provider

OutlookAddressbook

  • Type: string
  • Required: Yes
  • Description: The name of the Outlook address book to access
  • Common Values:
    • "Contacts" - Your personal contacts folder
    • "Global Address List" - Organization's GAL (Exchange environments)
    • Custom address book names configured in Outlook

ConnectionSettings

  • Type: string
  • Description: Reserved for future use (currently unused but required)
  • Default: ""

SQLQuery

  • Type: string
  • Description: Reserved for future filtering capabilities
  • Default: ""

Optional Properties

Timeout

  • Type: integer
  • Default: 30
  • Description: Maximum time in seconds to wait for Outlook operations

FieldAliasOrder

  • Type: Dictionary<string, string>
  • Description: Maps internal field names to friendly display names
  • Available Fields:
    • FullName - Complete name from address book
    • FirstName - First name (if available)
    • LastName - Last name (if available)
    • EmailAddress - Email address
    • CompanyName - Company/organization name
    • JobTitle - Position/title
    • BusinessPhone - Business telephone number
    • MobilePhone - Mobile phone number
    • BusinessAddress - Business address
    • Type - Entry type (Contact, Exchange User, etc.)

UseLocalCache

  • Type: boolean
  • Default: false
  • Description: Enable caching to improve performance for large address books

LocalCachePath

  • Type: string
  • Description: Path for cached data (supports environment variables)
  • Example: "%temp%\\outlook_cache"

Example 1: Personal Contacts

{

    "$connectionType": "Outlook",
    "OutlookAddressbook": "Contacts",
    "ConnectionSettings": "",

Finding Your Address Book Names

To find the exact names of your Outlook address books:

  1. Open Microsoft Outlook
  2. Click on Home tab > Address Book button (or press Ctrl+Shift+B)
  3. In the Address Book window, use the dropdown at the top
  4. The dropdown shows all available address book names - use these exact names in your configuration

Common address book names:

  • Contacts - Your personal Outlook contacts
  • Global Address List - Company directory (Exchange/Microsoft 365)
  • Suggested Contacts - Auto-collected contacts
  • Any custom address books you've created

Returned Data Structure

The Outlook provider returns a DataSet with the following columns:

Column Name Data Type Description
FullName string Complete display name
FirstName string First name (when available)
LastName string Last name (when available)
EmailAddress string Primary email address
CompanyName string Company/organization name
JobTitle string Job title/position
BusinessPhone string Business telephone number
MobilePhone string Mobile phone number
BusinessAddress string Business address
Type string Entry type identifier

Note: Not all fields may be populated for every contact, depending on what information is available in Outlook.

Troubleshooting

"Microsoft Outlook is not installed"

  • Ensure Microsoft Outlook is installed on the system
  • Verify that Outlook is properly licensed

"Address book 'X' not found"

  • Check the exact spelling of the address book name (case-sensitive)
  • Open Outlook's Address Book (Ctrl+Shift+B) to verify available names
  • Ensure you have permission to access the address book

"Access denied to Outlook"

  1. Open Outlook
  2. Go to File > Options > Trust Center > Trust Center Settings
  3. Select Programmatic Access
  4. Ensure appropriate settings allow programmatic access
  5. You may need to approve access when prompted

"No entries found in address book"

  • Verify the address book contains contacts
  • Check if you have read permissions for the address book
  • For Exchange GAL, ensure you're connected to the server

Performance Issues with Large Address Books

  • Enable UseLocalCache in configuration
  • Increase the Timeout value
  • Consider filtering contacts (future feature via SQLQuery)

Security Considerations

  • The add-in requires programmatic access to Outlook
  • Outlook may prompt users to grant access when first connecting
  • Cached data is stored locally - ensure appropriate file permissions
  • Consider data privacy when using Global Address Lists
  • Cached data is not encrypted - use secure storage locations

Best Practices

  1. Use Descriptive Dataset Names: Name datasets clearly (e.g., "ClientContacts", "StaffDirectory")
  2. Enable Caching for Large Lists: Use local cache for GAL or large contact lists
  3. Map Fields Appropriately: Use FieldAliasOrder to provide user-friendly column names
  4. Set Appropriate Timeouts: Increase timeout for large address books or slow networks
  5. Test with Small Address Books First: Verify configuration with personal contacts before using GAL

Future Enhancements

The following features are planned for future releases:

  • SQLQuery Support: Filter contacts based on criteria
  • Incremental Updates: Only fetch changed contacts
  • Custom Field Mapping: Support for custom Outlook contact fields
  • Multiple Profile Support: Specify which Outlook profile to use
