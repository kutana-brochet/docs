K+B Office Add-ins

Ribbon

The file Ribbon.xml is used for the configuration of all the buttons on the Ribbon.

Where this config file is located

There can be a ribbon.xml file for each Office applications.

There needs to be aseparate Ribbon.xml file for each application which is located in the following locations:

%appdata%\Kutana+Brochet\<OfficeApp Name>\config

eg ...\AppData\Roaming\Kutana+Brochet\Word\config

or

%localappdata%\Kutana+Brochet\<OfficeApp Name>\config

The file located in the %localappdata% (..\Local\..) will take precedent on the one in %appdata% (..\Roaming\..). In this situation the file in Local will be used and the file in Roaming will be ignored.

(see Using Configuration layering for the configuration files for more details on laying)

XML Schema

This file is written in XML and conforms the standard Microsoft Office Ribbon XML schema.

The integration with Kutana+Brochet is through actions using the following five element parts:

  1. id
  2. image
  3. onAction
  4. getVisible
  5. getEnabled

The Kutana+Brochet Coms addin contains a set of built in images. The list of images available can be seen in the following GitHub link: https://github.com/kutana-brochet/office-addins/tree/master/src/VSTO/WordAddIn/Resources

Example style Button

Below is the XML for a button on the ribbon for applying the style Level 1 Heading to the selected text:

<button id="OperativeLevel1Heading"
    image="Level 1 Heading"
    onAction="RunScript"
    label="Level 1 Heading" showLabel="false"
    getVisible="GetVisibleByScript"
    getEnabled="GetEnabledByScript" 
    screentip="Level 1 Heading - "
    supertip="Applies the first level heading style eg 1. in bold and ALLCAPS." />

Notes on each item:

  1. id="OperativeLevel1Heading" - this is only relevant when the onAction is "RunScript".

This is used by the configuration setting "ScriptConfiguration" "Mappings"to define the commands to be run and when the button if visible and enabled.

NB: The id needs to be unique for each ribbon element. To have two buttons that run the same script there will need to be a Mapping for each id but these can run the same Script.

  1. image="Level 1 Heading" - this is used to define the image to be used and displayed on the ribbon.

The image can either be an inbuilt resource in the COMS addin or one generated by the system when Word is loaded.

See image below

https://github.com/kutana-brochet/office-addins/tree/master/src/VSTO/WordAddIn/Resources

  1. onAction="RunScript" - defines the command to be run.

See onAction below for the for list of available commands

  1. getVisible - determines when the ribbon element is visible. Available callbacks:
Callback Used With Matches Against
GetVisibleByScript Script buttons (onAction="RunScript") ScriptConfiguration.Mappings by button id
GetVisibleById Any ribbon element RulesConfiguration.RibbonItemRules by element id
GetVisibleByTag Any ribbon element RulesConfiguration.RibbonItemRules by element tag

See Script Configuration for script-based visibility. See Ribbon Item Rules for rules-based visibility.

  1. getEnabled - determines when the ribbon element is clickable. Available callbacks:
Callback Used With Matches Against
GetEnabledByScript Script buttons (onAction="RunScript") ScriptConfiguration.Mappings by button id
GetEnabledById Any ribbon element RulesConfiguration.RibbonItemRules by element id
GetEnabledByTag Any ribbon element RulesConfiguration.RibbonItemRules by element tag

See Script Configuration for script-based enabled logic. See Ribbon Item Rules for rules-based enabled logic.
