Utilities

The Utilities group contains a set of tools designed to make document management and formatting more efficient and consistent. These tools help you maintain style standards, manage templates, and apply common settings with just a few clicks, ensuring that your documents remain professional and aligned with your organization's formatting rules.

NB: This is a standard Word idMso command

Toggle Paragraph Marks (idMso ParagraphMarks)

Known as "Show/Hide ¶" in native Word, the Toggle Paragraph Marks feature in Word, often represented by the ¶ symbol, is used to show or hide non-printing characters in your document. These characters are invisible when you print or view the document normally but can affect spacing, layout, and formatting.

What it shows:

When you toggle it on, Word displays symbols for:

  • Paragraph breaks (¶): Marks the end of a paragraph.
  • Spaces (·): Small dots showing spaces between words.
  • Tabs (→): Arrows showing where tab stops are used.
  • Line breaks (↵): Indicates a manual line break without starting a new paragraph.
  • Hidden text: Text formatted as hidden will also appear.

Why it’s useful:

  • Troubleshooting formatting issues: If spacing looks off or alignment isn’t right, these markers help you see exactly what’s happening.
  • Editing complex documents: Shows where extra spaces, tabs, or paragraph marks might be affecting layout.
  • Consistent formatting: Ensures that elements like tables, lists, and indents are structured correctly.

see more at https://support.microsoft.com/en-gb/office/show-or-hide-tab-marks-in-word-84a53213-5d02-404a-b022-09cae1a3958b

Ribbon.xml

    <toggleButton idMso="ParagraphMarks" />

Clear Text Formatting (Command ClearTextFormatting)

The Clear Text Formatting tool removes all manual character-level formatting from the selected text.

Note: This is equivalent to pressing the Control+Spacebar shortcut key on the keyboard - but there is no in-built ClearFontFormatting button in Word to add to the Quick Access Toolbar!

This is useful when text has been copied from other sources or heavily edited and contains inconsistent font styling.

  • Strips away character-specific formatting, such as:
    • Font type, size, and colour
  • Bold, italic, underline, strikethrough
  • Highlighting and text effects
  • Leaves the plain text content intact

This is often used in conjuction with the Clear Paragraph Formatting tool.

Note: this does not remove colour applied using the "Text Highlight Colour" tool - this is by design (by Microsoft), Word considers it a display option.

Usage

The tool only applies to the current selected text - if only part of a paragraph is selected then applies to the selected text ONLY.

Ribbon.xml

<button id="ClearTextFormatting"
        onAction="ClearTextFormatting"
        label="Clear Manual Text Formatting"
        showLabel="false"
        image="ClearTextMarking"
        screentip="Clear Manual Text Formatting"
        supertip="Clear additional manual text formatting. Any changes to the text other than that defined by the style will be removed." />

Note: that some older .xml files call a function RemoveTextFormatting - this will not work.

Clear Paragraph Formatting (Command ClearParagraphFormatting)

The Clear Paragraph Formatting tool removes all manual paragraph-level formatting from the selected text, restoring it to the formatting defined by the applied style.

Note: This is equivalent to pressing the Control+Q shortcut key on the keyboard - but there is no in-built ClearParagraphFormatting button in Word to add to the Quick Access Toolbar!

This is useful when text has inconsistent or unwanted formatting applied using the Paragraph Formatting options in Word.

  • Strips away paragraph-specific formatting, such as:
    • Indents (left, right, first line)
    • Line spacing and spacing before/after paragraphs
    • Alignment (left, right, center, justify)
    • Tabs and paragraph-level borders/shading
  • Keeps the text content and character formatting (like bold or italic) intact

This is often used in conjuction with the Clear Text Formatting tool.

Usage

The tool only applies to the current selection - if only part of a paragraph is selected then applies to the whole paragraph as this is paragraph formatting.

Ribbon.xml

<button id="ClearParagraphFormatting"
        onAction="ClearParagraphFormatting"
        label="Clear Paragraph Formatting"
        showLabel="false"
        image="ClearParagraphMarking"
        screentip="Clear Manual Paragraph Formatting"
        supertip="Clear additional manual paragraph formatting. Any changes to the paragraph other than that defined by the style will be removed." />

Note: that some older .xml files call a function RemoveTextFormattingAll - this will not work.

Toggle Navigation Pane (idMso DocumentMapReadingView)

The Navigation pane is a built-in tool in Microsoft Word that helps you quickly browse, search, and reorganise the content of your document. It’s especially useful in large or structured documents with headings.

  • Headings View
    • Displays a list of all headings in the document, based on styles (Heading 1, Heading 2, etc.)
    • Lets you jump directly to any section by clicking the heading
    • Supports drag-and-drop to reorder sections, which moves both the heading and its associated content
  • Page Thumbnails View
    • Shows small thumbnail previews of each page
    • Helps you visually navigate to specific pages quickly
  • Search (Results) View
    • Lets you search for words or phrases in the document
    • Displays a list of all results; clicking a result jumps to its location

Usage

The tool only applies to the current selected document. This command needs to be used on a per document basis.

Ribbon.xml

Note: The icon has been changed to the standard one we use in the ribbon

<button idMso="DocumentMapReadingView"
   showLabel="false"
   imageMso="RmsNavigationBar" />

Toggle Print/Draft View

Provides a convenient way to switch between Draft view and Print Layout view

  • Draft View in Microsoft Word is a document view designed to help you focus on writing and editing text content without being distracted by complex layout elements. It prioritizes speed and simplicity over precise page appearance. It also clearly displays the Paragraph styles used.
  • Print Layout view in Microsoft Word shows your document exactly as it will appear on the printed page. It is the default and most commonly used view, designed for working on the visual formatting and layout of your document.

Document Protection

Tools that modify document content (Delete Double Paragraphs, Delete Double Spaces, Smart Quotes) are automatically disabled when the document cannot be edited. This applies when:

  • The document has any form of Word protection applied (Review > Restrict Editing)
  • The document is opened in read-only mode or marked as final

If a tool appears greyed out, check the document's protection status in Word's Review tab.

Note: Documents opened in Protected View (e.g. downloaded from the internet) must have editing enabled via the yellow banner before the add-in can interact with them.

Smart Quotes

The Smart Quotes tool helps you manage quotation marks consistently throughout your document.

  1. Toggle Smart Quotes
    • A toggle button to turn Word’s Smart Quotes feature on or off globally.
    • When enabled, Word automatically converts straight quotes (") to typographic (curly) quotes (“”) as you type.
  2. Convert to Smart Quotes
    • Converts all quotation marks in the selected text to Smart Quotes.
    • Useful for cleaning up pasted text or documents created with straight quotes.
  3. Convert to Straight Quotes
    • Converts all quotation marks in the selected text to straight quotes (").
    • Useful for coding, data entry, or formatting for plain text environments.

Usage

The tool first processes the selected text and then prompts you to check the rest of the document. If it reaches the end, it will ask whether you want to start from the beginning. After completing the operation, it reports the total number of replacements made.

AttachTemplate

The Attach Template tool lets you review and modify the template attached to the current document. It provides options to redefine the default template and optionally reset the document’s styles to match that template.

Example dialog box

Currently attached template: - displays the name of the template that is currently attached the to the selected document.

Default template to attach: - the contents (the template name) of the custom document property TemplatePropertyName is displayed. The property TemplatePropertyName contains the Custom Document Property name that this information is saved into if not set then the default K+BTemplate is used. This setting is used by the command AttachTemplateOnOpen which checks the correct template is attached to the document on the DocumentOpen event.

List of available templates: This list the available templates. See How the templates to be displayed are found below for the file location and the templates that are listed.

The templates that are listed show three further bits of information:

  1. Version - this is picked up from the Custom Document properties of the document. This will use the following sequence of custom document properties read in the following order: K+BVersion, BrochetVersion, TemplafyVersion (need to double check that this name)
  2. Date - this is picked up from the Custom Document properties of the document. This will use the following sequence of custom document properties read in the following order: K+BDate, BrochetDate, TemplafyDate (need to double check that this name)
  3. Additional information - this is picked up from the Comments field on the document properties (NOTE: not K+BComments)

Highlighted template in the List of avilable templates: Highlights the template name which is currently attached to the document (e.g., Normal.dotm).

How the templates to be displayed are found

See the Scheme below. The following options can be set

SearchForTemplateName - default setting is Normal.dotm. This is the template name that is looked for in the following locations:

  1. User Templates then
  2. Workgroup Template. Once found this is the templates location that is used to source all the templates listed in the listbox on the dialog box.

If the template name is not found in either of the subdirectories then the default setting is used (Normal.dotm)

Which templates are displayed in the listbox

See the Scheme below. The following options can be set

IncludeFilter - default setting is "^.*\.dot[mx]\(". This uses regular expressions to determine the templates to be listed. Example: "^Test.*\\.dot[mx]\) will only list templates which start with the text "Test" eg Testing 1.dotx, Testing 2.dotx.

ExcludeFilter - default setting is "". This uses regular expressions to determine the templates to be excluded from the list. All temporary Word template files (ones that start with ~ are excluded as default).

Key User Features

User selects one template that is to be attached to the selected document (highlighted in the list)
User ticks/unticks the **Reset default template name** option
User ticks/unticks the **Reset document styles** option

ON PRESSING OK

**Always on pressing OK** sets the selected document to have the selected template as the attached template.
**Reset default template name** if ticked saves the selected template name into the custom document property defined by the property "TemplatePropertyName". The default setting is "K+BTemplate".
**Reset document styles** if ticked resets all styles in the document to match the attached template.

Default config JSON

"AttachTemplateConfiguration": {
        "SearchForTemplateName": "Normal.dotm",
        "IncludeFilter": "^.*\\.dot[mx]$",
        "ExcludeFilter": "",
        "TemplatePropertyName": "K+BTemplate"
    }

PROPERTY OPTIONS

  1. "SearchForTemplateName" - this is the name of the template the code searches for to determine which templates location to use eg User Templates or Workgroup Templates.
  2. "IncludeFilter" - this is a regular expression that will list the templates to be displayed in the listbox.
  3. "ExcludeFilter" - this is a regular expression that will list the templates NOT to be displayed in the listbox.
  4. "TemplatePropertyName" - this is the name of the custom document property that holds the default template name. This information is used on the ??? option which makes sure the correct template is attached to the document. This is useful for Templafy clients or when a document is sent outside the firm and is returned with the document attached to Normal.dotm.
