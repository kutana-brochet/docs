Starter Document

The Starter Document menu lets you create a new Word document based on an existing pre-filled document. Unlike the New Document menu which creates a blank document from a template, a starter document already contains content — headings, boilerplate text, tables, or other standard elements — giving you a head start.

How It Works

  1. Click the Starter Document drop-down button on the ribbon
  2. The add-in discovers starter document files from your configured folder(s)
  3. A menu appears listing available starter documents, optionally grouped by folder
  4. Click a starter document name to create a new editable copy

Word opens a new document with the starter document's content. The original starter document file is not modified.

If your organisation uses a Document Management System (DMS), a pre-profile dialog may appear after the document is created.

Starter Documents vs Templates

New Document Starter Document
Source file .dotx / .dotm template .docx / .docm / .dotx / .dotm document
New document content Blank (styles only) Pre-filled with content
Use case Starting from scratch with correct styles Starting with standard boilerplate
Template attached The selected template Automatically resolved from the file

Template Resolution

Each starter document has an attached template embedded in its file metadata. When you create a document from a starter, the add-in:

  1. Reads the attached template name from the starter document file
  2. Looks up the matching template in your configured template location
  3. Creates the new document with both the starter content and the resolved template

If the attached template cannot be found, the default template is used as a fallback.

Ribbon.xml

<dynamicMenu id="starterDocumentMenu"
             label="Starter Document"
             size="large"
             image="NewDocument"
             getVisible="GetVisibleById"
             getEnabled="GetEnabledById"
             getContent="GetStarterDocuments" />

NB: The id attribute must match a key in the FileProviderConfiguration.StarterDocument configuration section.

The optional getVisible and getEnabled attributes use Ribbon Item Rules to control whether the menu appears and whether it is clickable. If omitted, the menu is always visible and enabled.

Staleness Indicators

Starter document menu items display a colour-coded icon to indicate how recently the file was last modified:

  • Default (blue icon) - the file is current
  • Amber icon - the file has not been updated for a configured number of days
  • Red icon - the file has not been updated for a longer period

The screentip for each item always shows the last modified date in short date format.

This helps you identify starter documents that may contain outdated content and need review.

Configuring Staleness

Staleness thresholds are configured in the Style.Staleness section:

"Style": {
  "Staleness": {
    "Thresholds": [
      { "AfterDays": 30, "Icon": "PersonaStatusAway" },
      { "AfterDays": 90, "Icon": "PersonaStatusBusy" }
    ]
  }
}
Property Description
AfterDays Number of days since last modification before this threshold applies
Icon The Office icon to display (e.g. PersonaStatusAway for amber, PersonaStatusBusy for red)

Thresholds are evaluated in order; the last matching threshold wins.

Configuration

The Starter Document menu is configured within the FileProviderConfiguration section of your JSON configuration file.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "FileProviderConfiguration": {
    "StarterDocument": {
      "starterDocumentMenu": {
        "Source": {
          "Location": "%Workgroup%",
          "MaxDepth": 2,
          "Extensions": [".docx", ".docm", ".dotx", ".dotm"]
        },
        "Style": {
          "Title": "Starters",
          "Type": "Flat",
          "RemoveLeadingNumbersFromTemplateNames": true,
          "RemoveLeadingNumbersFromContainerNames": true,
          "SeparateByContainer": true,
          "Staleness": {
            "Thresholds": [
              { "AfterDays": 30, "Icon": "PersonaStatusAway" },
              { "AfterDays": 90, "Icon": "PersonaStatusBusy" }
            ]
          }
        }
      }
    }
  }
}

NB: The key "starterDocumentMenu" must match the id attribute on the <dynamicMenu> element in your Ribbon.xml.

Source Properties

Property Required Default Description
Location No %Workgroup% Root folder path. Supports %Workgroup% and %User% placeholders
MaxDepth No 0 Subdirectory depth to search. 0 = root only
Extensions No [".docx", ".docm", ".dotx", ".dotm"] File extensions to include
Prefixes No [] Only show files with these name prefixes. Empty = show all
ExcludedContainers No [] Folder names to skip during discovery

Style Properties

Property Required Default Description
Title No none Optional header text shown at the top of the menu
Type No Flat Flat (single list with separators) or Group (nested submenus)
RemoveLeadingNumbersFromTemplateNames No true Strip numeric prefixes from file names
RemoveLeadingNumbersFromContainerNames No true Strip numeric prefixes from folder names
SeparateByContainer No true In Flat mode, show folder name separators between groups
Staleness No none Staleness threshold configuration (see above)

Display Styles

Flat

Items appear as a single list. When SeparateByContainer is true, items from different folders are visually separated with a folder name header.

Group

Items are organised into submenus by folder (maximum 2 levels deep). Folders with only a single subfolder and no files are compacted into one level.

Troubleshooting

Menu is empty

Problem: The Starter Document drop-down shows no items.

Solutions:

  1. Verify the Location path exists and is accessible
  2. Check that files have the correct extensions (.docx, .docm, .dotx, or .dotm)
  3. Ensure MaxDepth is set high enough to reach your files
  4. Confirm the configuration key matches the ribbon button id

Staleness icons not appearing

Problem: All items show the default icon regardless of file age.

Solutions:

  1. Check that Staleness.Thresholds is configured in the Style section
  2. Verify that AfterDays values are appropriate for your update cycle
  3. Confirm the Icon values are valid Office icon names

Wrong template applied

Problem: A document created from a starter has incorrect styles.

Solutions:

  1. Open the starter document file and check its attached template via File > Info
  2. Ensure the attached template exists in your configured template location
  3. If the template cannot be resolved, the default template is used as a fallback
