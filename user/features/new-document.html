New Document

The New Document menu lets you create a new Word document based on an organisation template (.dotx or .dotm). Rather than navigating to a template folder manually, you select from a dynamically populated drop-down menu on the ribbon.

How It Works

  1. Click the New Document drop-down button on the ribbon
  2. The add-in discovers template files from your configured template folder(s)
  3. A menu appears listing all available templates, optionally grouped by folder
  4. Click a template name to create a new document based on that template

If your organisation uses a Document Management System (DMS), a pre-profile dialog may appear after the document is created, allowing you to save and categorise the document immediately.

Ribbon.xml

The New Document menu is a <dynamicMenu> element. Your organisation's Ribbon.xml should include an element like:

<dynamicMenu id="newDocumentMenu"
             label="New Document"
             size="large"
             image="NewDocument"
             getVisible="GetVisibleById"
             getEnabled="GetEnabledById"
             getContent="GetNewDocuments" />

NB: The id attribute must match a key in the FileProviderConfiguration.NewDocument configuration section. Multiple New Document menus can be configured with different IDs to show different template sets.

The optional getVisible and getEnabled attributes use Ribbon Item Rules to control whether the menu appears and whether it is clickable. If omitted, the menu is always visible and enabled.

Configuration

The New Document menu is configured within the FileProviderConfiguration section of your JSON configuration file.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "FileProviderConfiguration": {
    "NewDocument": {
      "newDocumentMenu": {
        "Source": {
          "Location": "%Workgroup%",
          "MaxDepth": 2,
          "Extensions": [".dotx", ".dotm"]
        },
        "Style": {
          "Title": "Templates",
          "Type": "Flat",
          "RemoveLeadingNumbersFromTemplateNames": true,
          "RemoveLeadingNumbersFromContainerNames": true,
          "SeparateByContainer": true
        }
      }
    }
  }
}

NB: The key "newDocumentMenu" must match the id attribute on the <dynamicMenu> element in your Ribbon.xml.

Source Properties

Property Required Default Description
Location No %Workgroup% Root folder path. Supports %Workgroup% and %User% placeholders
MaxDepth No 0 Subdirectory depth to search. 0 = root only
Extensions No [".dotx", ".dotm"] File extensions to include
Prefixes No [] Only show files with these name prefixes. Empty = show all
ExcludedContainers No [] Folder names to skip during discovery

Style Properties

Property Required Default Description
Title No none Optional header text shown at the top of the menu
Type No Flat Flat (single list with separators) or Group (nested submenus)
RemoveLeadingNumbersFromTemplateNames No true Strip numeric prefixes (e.g. 01_Letter becomes Letter)
RemoveLeadingNumbersFromContainerNames No true Strip numeric prefixes from folder names
SeparateByContainer No true In Flat mode, show folder name separators between groups

Path Placeholders

  • %Workgroup% - resolves to the shared workgroup template path (typically set via Group Policy or installer)
  • %User% - resolves to the user's personal template path

Display Styles

Flat

Items appear as a single list. When SeparateByContainer is true, items from different folders are visually separated with a folder name header.

Group

Items are organised into submenus by folder. Each folder becomes a submenu (maximum 2 levels deep). Folders with only a single subfolder and no files are compacted into one level (e.g. Corporate / Letters instead of a Corporate submenu containing a Letters submenu).

Template Organisation

File Naming

  • Templates can use numeric prefixes to control sort order: 01_Letter.dotx, 02_Memo.dotx
  • When RemoveLeadingNumbersFromTemplateNames is true, the prefix is stripped from the display name
  • Supported prefix formats: 01_, 01-, 01., 01 , or just 01

Folder Structure

  • Organise templates in subfolders for logical grouping
  • Set MaxDepth to control how deep the add-in searches
  • Use ExcludedContainers to hide folders like Archive or Deprecated

Troubleshooting

Menu is empty

Problem: The New Document drop-down shows no items.

Solutions:

  1. Verify the Location path exists and is accessible
  2. Check that template files have the correct extensions (.dotx or .dotm)
  3. Ensure MaxDepth is set high enough to reach your template files
  4. Confirm the configuration key matches the ribbon button id

Templates not appearing from subfolders

Problem: Templates in subfolders are not shown.

Solutions:

  1. Increase the MaxDepth value (default is 0 which only searches the root folder)
  2. Check that the subfolder is not listed in ExcludedContainers
