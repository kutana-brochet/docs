ScriptConfiguration

"$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json"

Description

The JSON element "ScriptConfiguration" is used for defining the "Scripts" that are run by ribbon buttons. These "Scripts" need to be linked back to the ribbon button by the "Mappings" section.

The two elements of "ScriptConfiguration" are:

  1. "Mappings"
  2. "Scripts"

Recommended JSON file name

It is recommended that the JSON file holding this information is called Scripts.json.

This section of the configuration can be divided between two separate files eg ScriptsStyleButtons.json and ScriptsMacroButtons.json and then will both be read and the information joined together. Remember if there is a "Mapping" that is defined in both files the "Mapping" in the filename that in later in the alapbetic list ScriptsStyleButtons.json will overwrite the other.

"Mappings"

This section is used for linking the ribbon elements (usually buttons) to the "Script" that will be run when the button is pressed.

The secion also controls the button visibilty by linking to the and "RulesConfiguration" "Workflows" logic of when the ribbon element (usually button) is visible in the ribbon.

Example

Below is an example of two Mappings for the Party 1 and Background 1 styles buttons:

XML from the Ribbon.xml file:

<box id="PartyBox1" boxStyle="vertical">
    <button id="PartyParties"
       image="Parties 1" 
       onAction="RunScript" 
       label="Parties" 
       showLabel="false" 
       getVisible="GetVisibleByScript" 
       getEnabled="GetEnabledByScript" />
    <button id="PartyBackground" 
       image="Background 1" 
       onAction="RunScript" 
       label="Background" 
       showLabel="false" 
       getVisible="GetVisibleByScript" 
       getEnabled="GetEnabledByScript" />
</box>

JSON from the configuration file Scripts.json

Note: the button id in the xml must match the name of the "Mappings" element name in the "Mappings" selction

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "ScriptConfiguration": {
    "Mappings": {
      "PartyParties": {
        "Script": "StyleParties",
        "EnabledWhen": "PartyGroupVisible",
        "VisibleWhen": "PartyGroupVisible"
      },
      "PartyBackground": {
        "Script": "StyleBackground",
        "EnabledWhen": "PartyGroupVisible",
        "VisibleWhen": "PartyGroupVisible"
      },

There are three options of the "Mappings" element:

  1. "Script" - this is the name of the Script which is defined in the "Scripts": section (usually below the Mappings section of the "ScriptConfiguration"

See the Scripts section below for a brief description. See the full documentation in Scripts.mp as there are a large number of options that can be used in the Scripts.

  1. "EnabledWhen" - this contains the name of the rules from "Rules" element of RulesConfiguration": { "Workflows": [{ "WorkflowName": "RibbonRules"

See the EnabledWhen section below for a brief description. See the full documentation of RulesConfiguration.mp for documentation.

  1. "VisibleWhen" - this contains the name of the rules from "Rules" element of RulesConfiguration": { "Workflows": [{ "WorkflowName": "RibbonRules"

See the EnabledWhen section below for a brief description. See the full documentation of RulesConfiguration.mp for documentation.

"Scripts"

The Scripts section is where the actions are defined when the ribbon button is pressed.

See the full documentation in Scripts.md

Example Scripts for the Party 1 and Background 1 buttons defined above:

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "ScriptConfiguration": {
    "Scripts": {
      "StyleParties": {
        "Comment": "Applies Parties Style",
        "Commands": [
          {
            "$type": "ApplyStyle",
            "Range": {
              "$type": "Selection",
              "SubRange": "All"
            },
            "Options": {
              "StyleName": "Parties"
            }
          }
        ]
      },      
      "StyleBackground": {
        "Comment": "Applies Background Style",
        "Commands": [
          {
            "$type": "ApplyStyle",
            "Range": {
              "$type": "Selection",
              "SubRange": "All"
            },
            "Options": {
              "StyleName": "Background"
            }
          }
        ]
      },
