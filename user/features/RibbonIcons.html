RibbonIcons

"$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json"

Description

The JSON element "RibbonIcons" is used for defining the icons that are auto-generated when Word loads.

The is one elements of "RibbonIcons":

  1. "Icons"

Recommended JSON file name

It is recommended that the JSON file holding this information is called StyleButtonsConfig.json.

This section of the configuration can be divided between two separate files eg StyleButtonsConfigA.json and StyleButtonsConfigB.json and then will both be read and the information joined together. Remember if there is a "Icons" that is defined in both files the "RibbonIcons" in the filename that in later in the alapbetic list StyleButtonsConfigB.json will overwrite the other.

"RibbonIcons"

This section is used for defining the icon image.

Example

Below is an example of two Mappings for the Party 1 and Background 1 styles buttons:

XML from the Ribbon.xml file:

<box id="PartyBox1" boxStyle="vertical">
    <button id="PartyParties"
       image="Parties 1" 
       onAction="RunScript" 
       label="Parties" 
       showLabel="false" 
       getVisible="GetVisibleByScript" 
       getEnabled="GetEnabledByScript" />
    <button id="PartyBackground" 
       image="Background 1" 
       onAction="RunScript" 
       label="Background" 
       showLabel="false" 
       getVisible="GetVisibleByScript" 
       getEnabled="GetEnabledByScript" />
</box>

JSON from the configuration file StyleButtonsConfig.json

Note: There are two settings for each icon the second being the same icon name except it has Dark appended to the name. This creates a black version of the same icons which is used when running Office in dark mode. Usually the ony difference between the two settings are the FontColour for example it is changed to White for dark mode.
