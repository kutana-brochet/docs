Scripts

Scripts allow you to automate multiple document editing operations with a single button click. Each script contains a sequence of commands that execute in order, helping you apply consistent formatting and maintain document standards efficiently.

What are Scripts?

Scripts are configurable automation tools that combine multiple editing commands into a single action. Rather than manually applying several formatting changes, you can create a script that applies all changes automatically to your selected text or entire document.

Benefits

  • Time-Saving: Execute multiple formatting operations with one click
  • Consistency: Ensure uniform formatting across documents
  • Reduced Errors: Eliminate manual formatting mistakes
  • Customisable: Configure scripts to match your organisation's standards
  • Reusable: Create once, use repeatedly across documents

How is a Script activated?

A Script can be activated via a button on the ribbon.

Example ribbon button XML:

<button id="OperativeLevel1Heading" 
  image="Level 1 Heading"
  onAction="RunScript" 
  label="Level 1 Heading" />

Notice that the onAction="RunScript" is being used. This is the command that is being run which will find the correct script to run using the button id="OperativeLevel1Heading".

There is a JSON element ScriptConfiguration that needs to be set with the button mapping and the script instructions.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "ScriptConfiguration": {
    "Mappings": {
      "OperativeLevel1Heading": {
        "Script": "StyleLevel1Heading",
        "EnabledWhen": "OperativeGroupFullVisible",
        "VisibleWhen": "OperativeGroupFullVisible"
      }
    },
    "Scripts": {
      "StyleLevel1Heading": {
        "Comment": "Applies Level 1 Heading Style",
        "Commands": [
          {
            "$type": "ApplyStyle",
            "Range": {
              "$type": "Selection",
              "SubRange": "All"
            },
            "Options": {
              "StyleName": "Level 1 Heading"
            }
          }
        ]
      }
    }
  }
}

Creating a Script element

A Script element is made up of three parts; a name and then two elements: Comment and Commands.

  • Name - this needs to be unique and matches the Script name in the Mappings section of the JSON file
  • "Comment": this element is only used for information by the person editting the JSON file as comments are not possible in JSON files.
  • "Commands": this element is an array of Script commands that are run in sequence. There must be at least one script command eg to apply a style or a sequence of commands to perform a sequence of commands.
"Scripts": {
  "StyleLevel1Heading": {
    "Comment": "Applies Level 1 Heading Style",
    "Commands": [
      {
        "$type": "ApplyStyle",
        "Range": {
          "$type": "Selection",
           "SubRange": "All"
        },
         "Options": {
          "StyleName": "Level 1 Heading"
        }
      }
    ]
  }

"Commands" elements

A Commands element is made up of three parts; a name and then two elements: Comment and Commands.

  • "$type" - this is the name of the script command. There are 12 types of command - Mandatory.
  • "Range": - this holds the range that the script is going to affect within the current Word document. Some of the Command types do not require a Range option (Mandatory for some types).
  • "Options": this element holds the different options available for the Command type. There is always at least one Options for all script commands.

Available Script Commands types

Scripts support 13 types of commands that can be combined in any sequence:

Formatting Commands "$type"

1. Apply Font Formatting ("$type": "ApplyFontFormatting")

Applies character-level formatting to text, including:

  • Bold, italic, underline, doublestrikethrough, strikethrough
  • CharacterCase
  • Font colour and colour index
  • All caps, small caps
  • Superscript and subscript
  • Text position
  • Hidden text

2. Apply Paragraph Formatting ("$type": "ApplyParagraphFormatting")

Applies paragraph-level formatting, including:

  • Alignment (left, right, centre, justify)
  • Indentation (left, right, first line)
  • Line spacing and spacing before/after
  • KeepWithNext, KeepWithNext, PageBreakBefore
  • OutlineLevel, NoLineNumber
  • Tab stops
  • WindowControl

3. Apply Style ("$type": "ApplyStyle")

Applies a named paragraph style to the selected text or range.

NB: There are three types of Style: Paragraph, Character and Linked (Paragraph and Character)

4. Clear manually applied font formatting ("$type": "ClearFontFormatting")

Clears all manually applied font formatting to the selection or range - equivalent to Ctrl+Spacebar

5. Clear manually applied paragraph formatting ("$type": "ClearParagraphFormatting")

Clears all manually applied paragraph formatting to the selection or range.

Text Manipulation Commands

6. Delete Characters ("$type": "DeleteCharacters")

Removes specified characters from the text. Useful for cleaning up unwanted characters or formatting marks.

7. Delete Words ("$type": "DeleteWords")

Removes specified characters from the text. Useful for cleaning up unwanted characters or formatting marks.

8. Enforce Case ("$type": "EnforceCase")

Converts text to a specific case:

  • Uppercase (ALL CAPS)
  • Lowercase (all lowercase)
  • Title Case (First Letter Capital)
  • Sentence case (First letter of sentence)

9. Insert Field ("$type": "InsertField")

Inserts a Word field (such as page numbers, dates, or document properties) at the specified location.

10. Insert Word ("$type": "InsertWord")

Inserts specified text at the cursor position or within a range. Can optionally prevent duplicate insertions.

Custom Document Property Manipulation Commands

11. Set the value of a Custom Document Property ("$type": "SetCustomProperty")

Used to set the value of a Custom Document Property. These can be used to show or hide buttons for groups on the ribbon. Use in tandem with UnsetCustomProperty

12. Delete a Custom Document Property ("$type": "DeleteCustomProperty")

Used to unset the value of a Custom Document Property. These can be used to show or hide buttons for groups on the ribbon. Use in tandem with SetCustomProperty

VBA Macro Commands

13. Run VBA Macro ("$type": "RunMacro")

Executes an existing VBA macro by name with optional parameters. This allows you to integrate existing VBA automation with your script configurations.

Use Cases:

  • Execute legacy VBA macros from ribbon buttons
  • Run document-specific macros stored in templates
  • Trigger complex automation that's already implemented in VBA

Using Scripts

Scripts appear as buttons in your Word ribbon. The visibility and availability of scripts are controlled by configuration and can be customised per user or organisation.

Executing a Script

  1. Select text (if the script applies to a selection) or position your cursor
  2. Click the script button in the ribbon
  3. The script executes all commands in sequence
  4. Changes are applied to your document
Tip

Scripts respect Word's undo stack. If you need to reverse script changes, press Ctrl+Z immediately after execution.

Script Behaviour

  • Sequential Execution: Commands execute in the order configured

  • Range Application: Each command can target different ranges:

    • Entire document ("CommandRange": "EntireDocument") - "THIS DOES NOT EXIST IN THE SCHEMEA AT THE MOMENT SO NOT AVAILABLE"
    • Current selection ("CommandRange": "Selection")
    • All paragraphs of selection ("CommandRange": "Paragraphs")
    • First paragraph of selection ("CommandRange": "FirstParagraph")
    • Last paragraph of selection ("CommandRange": "LastParagraph")

    SubRange Application: the following Range options can also have a SubRange ("CommandRange": "Selection") and :

    • Selection ("CommandRange": "Selection") with sub-ranges (first word, last word, all words, first sentence, last sentence, all sentences)

    • Current selection ("CommandRange": "Selection")

  • Error Handling: If a command fails, script execution stops, and you receive an error message

Configuration

  • Shared Configuration: %appdata%/kutana+brochet/Word/Config
  • User Configuration: %localappdata%/kutana+brochet/Word/Config

Script Structure

Each script requires:

  • Name: Unique identifier for the script
  • Description: Plain-language explanation of what the script does
  • Commands: List of operations to perform
  • Mapping: Controls visibility and availability in the ribbon

Configuration Example

{
  "Scripts": {
    "FormatLegalDocument": {
      "Comment": "Applies standard legal document formatting",
      "Commands": [
        {
          "$type": "ApplyStyle",
          "CommandRange": "AllParagraphs",
          "StyleName": "Legal Body Text"
        },
        {
          "$type": "ApplyFontFormatting",
          "CommandRange": "Selection",
          "SubRange": "All",
          "Bold": 0,
          "Italic": 0
        },
        {
          "$type": "DeleteCharacters",
          "CommandRange": "EntireDocument",
          "SubRange": "All",
          "Characters": "  "
        }
      ]
    }
  },
  "Mappings": {
    "formatLegalDoc": {
      "Script": "FormatLegalDocument",
      "EnabledWhen": "Always",
      "VisibleWhen": "Always"
    }
  }
}

Configuration Options

EnabledWhen

  • Always: Script is always available
  • Custom rules expression (advanced)

VisibleWhen

  • Always: Script button always visible in ribbon
  • Custom rules expression (advanced)

CommandRange Options (Is the attribute mandatory ??? Check with Phill

  • EntireDocument: Applies to whole document ("CommandRange": "EntireDocument")
  • Selection: Applies to currently selected text ("CommandRange": "Selection")
  • Paragraphs: Applies to every paragraph in the selection ("CommandRange": "Paragraphs")
  • FirstParagraph: Applies to first paragraph only in the selection ("CommandRange": "FirstParagraph")
  • LastParagraph: Applies to last paragraph only in the selection ("CommandRange": "LastParagraph")

SubRange Options (for granular control)

  • All: Entire range
  • FirstWord: First word only
  • LastWord: Last word only Available on InsertText only
  • Start: First word only
  • End: Last word only

Command Types Reference

1. ApplyFontFormatting

Mandatory Properties:

  • Ranges:
    • $type: EntireDocument, FirstParagraph, LastParagraph, Paragraphs, Selection
    • SubRange: All, FirstWord, LastWord
  • Options: There needs to be at least one option for the Script to do anything

NB: EntireDocument applies to the Main story in the document - it dose not include text in a textbox in the main story

NB: The ("$type": "ClearFontFormatting") can be used to remove all manual formatting prior to running font formatting commands

"Comment": "Testing Applying Bold",
"Commands": [
  {
    "$type": "ApplyFontFormatting",
    "Range": {
      "$type": "Selection",
      "SubRange": "All"
    },
    "Options": {
      "Bold": "True"
    }
  } 
]

Available Options: at least one must be used for the script to do anything:

  • Bold, Italic: False, Toggle, True
  • CharacterCase: FullWidth, HalfWidth, LowerCase, NextCase, TitleSentence, TitleWord, ToggleCase, UpperCase
  • AllCaps, SmallCaps: False, Toggle, True
  • Underline: None, Single, Double, Dotted, etc.
  • Colour: Text string that can either be the RGB colour value enters as a text string eg RGB(0,0,255) or the hex value eg #1234AB
  • ColourIndex: 0 - Not sure how this is used ?????
  • Hidden: False, Toggle, True
  • Position: Raise/lower text (in points)
  • Subscript, Superscript: False, Toggle, True
  • StrikeThrough, DoubleStrikeThrough: False, Toggle, True

2. ApplyParagraphFormatting

Mandatory Properties:

  • Ranges:
    • $type: EntireDocument, FirstParagraph, LastParagraph, Paragraphs, Selection
  • Options: There needs to be at least one option for the Script to do anything

NB: There is no SubType option in the Range for ApplyParagraphFormatting as it applies to whole paragraphs.

"Comment": "Testing removing space after",
"Commands": [
  {
    "$type": "ApplyParagraphFormatting",
    "Range": {
      "$type": "Selection"
    },
    "Options": {
      "SpaceAfter": 0 
    }
  } 
]

Available Options at least one must be used for the script to do anything:

  • Alignment: Center, Justified, Left, Right
  • BaseLineAlignment: Baseline, Center, FarEast50, Top
  • CharacterUnitFirstLineIndent, CharacterUnitLeftIndent, CharacterUnitRightIndent: 0 value in points (always +ve?)
  • FirstLineIndent: 0 value in points (always +ve?)
  • KeepTogether, KeepWithNext: False, True, Undefined
  • LeftIndent: 0 value in points (always +ve?)
  • LineSpacing: 0 value in points (always +ve?)
  • LineSpacingRule: AtLeast, Double, Exactly, Multiple, OnePointFive, Single
  • LineUnitAfter, LineUnitBefore: 0 value in points (always +ve?)
  • MirrorIndents: False, True, Undefined
  • NoLineNumber: False, True, Undefined
  • OutlineLevel: BodyText, Level1, Level2, . . . Level9
  • PageBreakBefore: False, True, Undefined
  • RightIndent: 0 value in points (always +ve?)
  • SpaceAfter, SpaceBefore: 0 value in points (always +ve?)
  • WidowControl: False, True, Undefined
