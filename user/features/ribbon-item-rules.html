Ribbon Item Rules

Ribbon Item Rules let you control which ribbon buttons, menus, and groups are visible or enabled. Rules are evaluated dynamically — when conditions change (e.g. a document is opened or a licence is activated), the ribbon updates automatically.

How It Works

  1. Each ribbon element has an id (or tag) attribute in Ribbon.xml
  2. The add-in matches that value against configured RibbonItemRules patterns
  3. If a matching rule is found, the rule's VisibleWhen and EnabledWhen values determine the item's state
  4. If no rule matches, the item defaults to visible and disabled

Configuration

Ribbon Item Rules are configured in the RulesConfiguration section of your JSON configuration file.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "RulesConfiguration": {
    "RibbonItemRules": [
      {
        "Pattern": "newDocumentMenu",
        "VisibleWhen": "Always",
        "EnabledWhen": "DocumentMenusEnabled"
      },
      {
        "Pattern": "starterDocument*",
        "VisibleWhen": "Always",
        "EnabledWhen": "DocumentMenusEnabled"
      },
      {
        "Pattern": "*",
        "VisibleWhen": "Always",
        "EnabledWhen": "Always"
      }
    ]
  }
}

Rule Properties

Property Required Default Description
Pattern Yes — Glob pattern matched against the ribbon item's id or tag
EnabledWhen No Always Workflow name determining when the item is enabled
VisibleWhen No Always Workflow name determining when the item is visible

Pattern Matching

Patterns use glob syntax:

Pattern Matches
newDocumentMenu Exact match only
starter* Any id starting with "starter"
*Snippet* Any id containing "Snippet"
* All ribbon items

Rules are evaluated in order — the first matching rule wins. Place specific patterns before broad ones.

VisibleWhen and EnabledWhen Values

Value Meaning
Always Always visible / enabled (this is the default)
(empty or omitted) Same as Always
(workflow name) Delegates to a RulesConfiguration.Workflows rule for dynamic evaluation

When using a workflow name, the corresponding workflow must be defined in the Workflows section of RulesConfiguration. See Script Configuration for details on configuring workflows.

Built-in Rule Conditions

The following conditions are available for use in workflow rule expressions:

Condition Expression Meaning
CanEdit context.CanEdit == true Document is open and editable (not protected or read-only)
HasDocument context.HasDocument == true A document is open
IsReadOnly context.Document.IsReadOnly == true Document is read-only

Document Protection and CanEdit

The CanEdit condition checks whether the active document can be modified. A document is not editable when:

  • Protected — the document has any form of Word protection applied (form fields only, tracked changes only, comments only, or read-only protection via Review > Restrict Editing)
  • Read-only — the document was opened in read-only mode, the file is marked read-only on disk, or the document has been marked as final

Buttons that modify document content (e.g. formatting, text replacement, field insertion) should use CanEdit as their EnabledWhen condition. This ensures they are automatically greyed out when the document cannot be edited.

Note: Documents opened in Protected View (e.g. files downloaded from the internet) are not visible to the add-in until the user clicks "Enable Editing" in the yellow banner. Once editing is enabled, the CanEdit condition evaluates normally.

Ribbon.xml Callbacks

To use Ribbon Item Rules, add getVisible and getEnabled attributes to your ribbon elements:

By ID (recommended for most elements)

<dynamicMenu id="newDocumentMenu"
             getVisible="GetVisibleById"
             getEnabled="GetEnabledById"
             getContent="GetNewDocuments" />

The id attribute value is matched against rule patterns.

NB: The getVisible="GetVisibleById" will also work on a group (do not include getEnabled="GetEnabledById" as this is not relvant for a group)

By Tag (for shared rules across elements)

<dynamicMenu id="insertSnippetMenu"
             getVisible="GetVisibleByTag"
             getEnabled="GetEnabledByTag"
             getContent="GetSnippetDocuments"
             tag="snippets" />

The tag attribute value is matched against rule patterns. This is useful when multiple ribbon elements should share the same visibility/enabled logic.

Available Callbacks

Callback Matches Against Default (no rule)
GetVisibleById control.Id visible
GetEnabledById control.Id disabled
GetVisibleByTag control.Tag visible
GetEnabledByTag control.Tag disabled

Default Behaviour

When no matching rule is found for a ribbon item:

  • Visibility defaults to visible — items do not disappear unexpectedly
  • Enabled defaults to disabled — items require an explicit rule to become clickable

This is intentional: it is safer for an item to appear greyed-out than to silently vanish from the ribbon.

Comparison with Script Button Rules

Script buttons (using onAction="RunScript") have their own visibility/enabled system via ScriptConfiguration.Mappings:

Ribbon Item Rules Script Mappings
Callbacks GetVisibleById / GetEnabledById GetVisibleByScript / GetEnabledByScript
Configuration RulesConfiguration.RibbonItemRules ScriptConfiguration.Mappings
Pattern matching Glob patterns (e.g. starter*) Exact id match only
Use case Dynamic menus, groups, non-script buttons Script-driven buttons

Both systems reference the same RulesConfiguration.Workflows for conditional logic. Choose Ribbon Item Rules for menus and groups; use Script Mappings for buttons that run scripts.

Examples

Hide a menu entirely

{
  "RibbonItemRules": [
    {
      "Pattern": "starterDocumentMenu",
      "VisibleWhen": "NeverVisible"
    }
  ]
}

Where NeverVisible is a workflow that always evaluates to false.

Enable menus only when a document is open

{
  "RibbonItemRules": [
    {
      "Pattern": "newDocumentMenu",
      "EnabledWhen": "DocumentIsOpen"
    },
    {
      "Pattern": "starterDocumentMenu",
      "EnabledWhen": "DocumentIsOpen"
    }
  ]
}

Catch-all rule for all ribbon items

{
  "RibbonItemRules": [
    {
      "Pattern": "newDocumentMenu",
      "EnabledWhen": "SpecialCondition"
    },
    {
      "Pattern": "*",
      "VisibleWhen": "Always",
      "EnabledWhen": "Always"
    }
  ]
}

The catch-all * rule ensures any item without a specific rule is visible and enabled. Place it last.

Troubleshooting

Menu is visible but greyed out

Problem: A ribbon item appears but cannot be clicked.

Solutions:

  1. Check that the item has an EnabledWhen rule configured (the default is disabled when no rule matches)
  2. Verify the referenced workflow exists in RulesConfiguration.Workflows
  3. Use "EnabledWhen": "Always" to make the item always enabled

Menu has disappeared

Problem: A ribbon item is not shown on the ribbon.

Solutions:

  1. Check that the item's VisibleWhen rule does not evaluate to false
  2. Verify the Pattern matches the element's id (or tag if using ByTag callbacks)
  3. Confirm the RulesConfiguration JSON is valid and loaded correctly
  4. Remember that rules are evaluated in order — an earlier catch-all pattern may be matching first

Rules not updating

Problem: Changing conditions (e.g. opening a document) does not update the ribbon.

Solutions:

  1. The ribbon refreshes automatically when the rules configuration changes via hot-reload
  2. If the issue persists, click the Refresh button (if available) or restart Word
