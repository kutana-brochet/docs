Insert Snippet

The Insert Snippet menu lets you insert the content of a pre-configured Word document into your active document at the current cursor position. This is useful for inserting standard clauses, boilerplate paragraphs, signature blocks, or any reusable content.

How It Works

  1. Place your cursor where you want the content inserted
  2. Click the Insert Snippet drop-down button on the ribbon
  3. The add-in discovers snippet files from your configured folder(s)
  4. A menu appears listing available snippets, optionally grouped by folder
  5. Click a snippet name to insert its content at the cursor position

NB: Unlike New Document and Starter Document, Insert Snippet does not create a new document. It inserts content into the currently active document.

Ribbon.xml

<dynamicMenu id="insertSnippetMenu"
             label="Insert Snippet"
             size="large"
             image="NewDocument"
             getVisible="GetVisibleByTag"
             getEnabled="GetEnabledByTag"
             getContent="GetSnippetDocuments"
             tag="insertSnippets" />

NB: The id attribute must match a key in the FileProviderConfiguration.InsertSnippet configuration section.

The optional getVisible and getEnabled attributes use Ribbon Item Rules to control whether the menu appears and whether it is clickable. When using the ByTag variants, the tag attribute value is matched against rules instead of the id. If omitted, the menu is always visible and enabled.

Configuration

The Insert Snippet menu is configured within the FileProviderConfiguration section of your JSON configuration file.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "FileProviderConfiguration": {
    "InsertSnippet": {
      "insertSnippetMenu": {
        "Source": {
          "Location": "%Workgroup%",
          "MaxDepth": 2,
          "Extensions": [".docx", ".docm"]
        },
        "Style": {
          "Title": "Snippets",
          "Type": "Flat",
          "RemoveLeadingNumbersFromTemplateNames": true,
          "RemoveLeadingNumbersFromContainerNames": true,
          "SeparateByContainer": true
        }
      }
    }
  }
}

NB: The key "insertSnippetMenu" must match the id attribute on the <dynamicMenu> element in your Ribbon.xml.

Source Properties

Property Required Default Description
Location No %Workgroup% Root folder path. Supports %Workgroup% and %User% placeholders
MaxDepth No 0 Subdirectory depth to search. 0 = root only
Extensions No [".docx", ".docm"] File extensions to include
Prefixes No [] Only show files with these name prefixes. Empty = show all
ExcludedContainers No [] Folder names to skip during discovery

Style Properties

Property Required Default Description
Title No none Optional header text shown at the top of the menu
Type No Flat Flat (single list with separators) or Group (nested submenus)
RemoveLeadingNumbersFromTemplateNames No true Strip numeric prefixes from file names
RemoveLeadingNumbersFromContainerNames No true Strip numeric prefixes from folder names
SeparateByContainer No true In Flat mode, show folder name separators between groups

Creating Snippet Files

Snippet files are standard Word documents (.docx or .docm) containing the content you want to insert.

Best Practices

  • One snippet per file: Each file should contain a single reusable block of content
  • Use styles: Format content using your organisation's styles so it integrates cleanly with the target document
  • Avoid headers/footers: Only the document body content is inserted
  • Use numeric prefixes for ordering: 01_Signature Block.docx, 02_Confidentiality Clause.docx

Folder Organisation

Organise snippets into subfolders by category:

Snippets/
  01_Clauses/
    Confidentiality.docx
    Non-Compete.docx
  02_Blocks/
    Signature Block.docx
    Address Block.docx
  03_Standard Text/
    Disclaimer.docx
    Copyright Notice.docx

With MaxDepth set to 1 or higher and SeparateByContainer enabled, the menu will group items by folder with separator headers.

Display Styles

Flat

Items appear as a single list. When SeparateByContainer is true, items from different folders are visually separated with a folder name header.

Group

Items are organised into submenus by folder (maximum 2 levels deep). Folders with only a single subfolder and no files are compacted into one level.

Troubleshooting

Menu is empty

Problem: The Insert Snippet drop-down shows no items.

Solutions:

  1. Verify the Location path exists and is accessible
  2. Check that snippet files have the correct extensions (.docx or .docm)
  3. Ensure MaxDepth is set high enough to reach your snippet files
  4. Confirm the configuration key matches the ribbon button id

Content not inserted

Problem: Clicking a snippet does nothing.

Solutions:

  1. Ensure you have an active document open - Insert Snippet requires a document to insert into
  2. Check that the snippet file still exists at its original location
  3. Verify that the add-in has a valid licence with the WordStandardTools entitlement

Formatting looks wrong after insert

Problem: Inserted content does not match the target document's formatting.

Solutions:

  1. Ensure the snippet file uses named styles rather than direct formatting
  2. Check that the target document has matching style definitions
  3. Consider using the Clear Text Formatting utility after insertion if needed
