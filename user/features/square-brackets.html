Square Brackets

The Square Brackets group provides tools for working with bracket pairs [...] in legal documents. These tools help you quickly navigate between placeholders, add bracketed sections, and manage bracketed content during document drafting and review.

Square brackets are commonly used in legal documents to mark:

  • Optional clauses or alternative text
  • Placeholder content to be filled in
  • Sections requiring review or client input
  • Draft content that may be removed

Features

The Square Brackets group includes six ribbon buttons organised into three functional areas:

Navigation

  • Find Next - Jump to the next square bracket pair
  • Find Previous - Jump to the previous square bracket pair

Adding Brackets

  • Add Brackets - Wrap the selected text in square brackets
  • Insert Blob - Insert a special marker between brackets (e.g., [X] or [●])

Removing Brackets

  • Delete Brackets - Remove the brackets but keep the content
  • Delete All - Remove both the brackets and their content

How It Works

Find Next / Find Previous

Navigate through all square bracket pairs in your document sequentially.

  1. Click Find Next or Find Previous on the ribbon
  2. The add-in locates the next/previous bracket pair from your current cursor position
  3. The bracket pair is highlighted with the content selected
  4. At the end of the document, you'll be prompted to restart from the beginning (or vice versa)

NB: If no bracket pairs exist in the document, you'll see a notification message.

Special handling for placeholder content controls:

  • If your cursor is inside a placeholder content control that displays brackets, the find operation will treat it as a bracket pair
  • This allows consistent navigation across both text brackets and content control placeholders

Add Brackets

Wrap your selected text in square brackets.

  1. Select the text you want to bracket
  2. Click Add Brackets on the ribbon
  3. Square brackets are added around your selection: Hello World → [Hello World]
  4. The selection is expanded to include the newly added brackets

NB: Works on any text selection, including:

  • Single words
  • Multiple paragraphs
  • Text that already contains brackets (creates nested brackets: [already [bracketed]])
  • Empty selection (inserts [] at cursor position)

Insert Blob

Insert a special marker between square brackets, commonly used to mark a position for later completion.

  1. Select the text you want to replace (or click where you want to insert)
  2. Click Insert Blob on the ribbon
  3. The selection is replaced with your configured blob text between brackets

Default behaviour:

  • With no configuration: Inserts [] with a Wingdings character
  • With configured text: Inserts [X] (or your configured marker)

NB: The blob character typically uses special formatting (e.g., Wingdings font) to create a visual marker like [●] that stands out in the document.

Delete Brackets

Remove the square brackets but preserve the content inside.

  1. Select a bracket pair (must include both [ and ])
  2. Click Delete Brackets on the ribbon
  3. The brackets are removed: [Hello World] → Hello World
  4. The cursor is positioned where the opening bracket was

Requirements:

  • Selection must start with [ and end with ]
  • If the selection is invalid, you'll see a notification message
  • The document remains unchanged if selection criteria aren't met

Delete All

Remove both the square brackets and their content.

  1. Select a bracket pair (must include both [ and ])
  2. Click Delete All on the ribbon
  3. The entire bracketed section is removed: Hello [World] Test → Hello Test
  4. The cursor is positioned where the opening bracket was

Requirements:

  • Selection must start with [ and end with ]
  • If the selection is invalid, you'll see a notification message
  • The document remains unchanged if selection criteria aren't met

NB: The id attributes can be customised as needed. The onAction attribute values must match the method names in ProductRibbon.cs.

The optional getVisible and getEnabled attributes use Ribbon Item Rules to control whether buttons appear and whether they are clickable.

Configuration

Square Brackets functionality can be customised via the SquareBracketsConfiguration section of your JSON configuration file.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "SquareBracketsConfiguration": {
    "Delimiter": {
      "Open": "[",
      "Close": "]"
    },
    "Blob": {
      "Text": "X",
      "FontName": "Wingdings",
      "FontSize": 10
    }
  }
}

Delimiter Properties

Property Required Default Description
Open No [ The opening delimiter character
Close No ] The closing delimiter character

NB: While you can customise the delimiter characters (e.g., to use curly braces {}), changing them affects all Square Brackets operations. Most legal workflows use the default square brackets.

Blob Properties

Property Required Default Description
Text No Empty string The text to insert between brackets. If empty, uses Wingdings character
FontName No Wingdings Font name for the blob character
FontSize No 10 Font size for the blob character

Common blob text values:

  • X - Simple letter X marker
  • Empty string - Uses Wingdings font for special character
  • ● - Bullet point (if font supports it)
  • TODO - Text reminder

Schema

See CoreConfiguration.schema.json for the full configuration schema. Add "$schema": "path/to/CoreConfiguration.schema.json" to your JSON file for IntelliSense and validation.

Troubleshooting

Find commands do nothing

Problem: Clicking Find Next/Previous has no effect.

Solutions:

  1. Verify square brackets exist in the document
  2. Check that you're using the correct bracket characters [ and ]
  3. Ensure the add-in has a valid licence

Delete commands show "invalid selection"

Problem: Delete Brackets or Delete All shows a notification about invalid selection.

Solutions:

  1. Ensure your selection starts with [ and ends with ]
  2. Select the entire bracket pair, not just the content inside
  3. Avoid selecting text before the opening bracket or after the closing bracket

Quick selection tip: Use Find Next/Previous to automatically select valid bracket pairs, then apply Delete operations.

Insert Blob shows unexpected character

Problem: Insert Blob creates a different character than expected.

Solutions:

  1. Check your Blob.Text configuration setting
  2. Verify the Blob.FontName is installed on your system
  3. Try using a standard font (e.g., Arial) with explicit text (e.g., X)

Add Brackets inserts at wrong position

Problem: Add Brackets doesn't wrap the intended text.

Solutions:

  1. Ensure you have text selected before clicking Add Brackets
  2. If inserting at a specific position, click to place your cursor first
  3. Verify no content controls or fields are interfering with the selection

Use Cases

Legal Drafting

Scenario: Marking optional clauses during contract drafting.

  1. Select the optional clause text
  2. Click Add Brackets to mark it as [optional clause]
  3. Use Find Next/Previous during review to navigate between all optional sections
  4. Use Delete Brackets to keep finalised clauses or Delete All to remove rejected ones

Document Review

Scenario: Marking placeholders for client input.

  1. Place cursor where information is needed
  2. Click Insert Blob to add [X] or [●] as a visual marker
  3. Send to client with instructions to replace bracketed markers
  4. Use Find Next to quickly locate and verify all inputs upon return

Template Preparation

Scenario: Creating document templates with multiple placeholders.

  1. Draft the template with descriptive placeholder text
  2. Select each placeholder section and click Add Brackets
  3. Use Find Next/Previous to review all bracketed sections
  4. Save as a template for future use

Document Finalisation

Scenario: Cleaning up a document before final delivery.

  1. Use Find Next to locate all remaining square brackets
  2. For sections to keep: Use Delete Brackets to remove markers
  3. For sections to remove: Use Delete All to delete entirely
  4. Continue until no brackets remain
