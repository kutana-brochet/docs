How to switch between different DMS options

The K+B Word msi can integrate with different Document Management Systems (DMS).

This is done using JSON configuration settings. For the DMS setting the JSON top level property "DocumentManagement": is used.

{
  "$schema": "https://kutana-brochet.github.io/schemas/7.0/CoreConfiguration.schema.json",
  "DocumentManagement": {

  }
}

NB: "Provider" is the proptery for defining the DMS type

Available DocumentManagement properties

  • CreateOptions: this is used for debug testing
  • PreProfileTemplates: (array) this holds the list of Templates to be pre-profiled (Front-end profiling). Each item can have the following properties:
    • CloseOnCancel (Optional)
    • DocType (Optional)
    • Filter (Mandatory)
  • Properties: ???What is this"
  • PropertyMappings: contains the following 3 properties allowing information to be saved into the Word document:
    • CustomProperties
    • DocumentVariables
    • DocumentProperties
  • Provider: iManage, NetDocuments and None

"PreProfileTemplates" properties

This propery is used for listing the Word templates that are to be pre-profiled (Front-end profile).

  • Filter: (Mandatory) this hold the name of the templates to be pre-profiled. It uses regular expressions.
  • CloseOnCancel: true or false If set to true and the Cancel button is pressed on the DMS SaveAs screen then the initially created document is CLOSED. The default setting is false.
  • DocType: used to pass to the DMS the document type to be set in the DMS SaveAs screen (currently only works in iManage - known bug in netDocs that information cannot be set in the SaveAs screen).

**Example: ** A template called Letter is pre-profiled and if the cancel button is pressed then the initial document is closed. Any template that starts with "AB Company" is pre-profiled and if the cancel button is pressed then the document is not closed and the rest of the process (Filesite-New) is continued.

"PreProfileTemplates": [
  {
    "Filter": "Letter",
    "CloseOnCancel": true
  },
  {
    "Filter": "AB Company*",
    "CloseOnCancel": false
  }
],

"PropertyMappings" properties

This property is used for defining the DMS information that is written into the Word document as either: Custom Document Properties, Variables or Document Properties.

The three different properties are: CustomProperties, DocumentVariables and DocumentProperties.

Each of these properties contain a list of paired values: "Name of variable": {{ DMSKeyName(s) and format }}

"DocumentVariables": {
    "DMSAuthor": "{{ author.value }}",
    "DMSOurRef": "{{ profileInfo.identifier }}v{{ profileInfo.version | string.to_int | object.format \"d2\" }}"
  },

There can be multiple DMSKeyNames with additional joining text. The DMSKeyNames can be formatted using SCRIBAN formatting (https://github.com/scriban/scriban).

Example: makes the document version into a 2 digit number but firstly converting the version string into an integer and then formatting this inro a 2 digit decimal number.

"DMSOurRef": "{{ profileInfo.identifier }}v{{ profileInfo.version | string.to_int | object.format \"d2\" }}"

"Provider" properties

This property is used to set the current DMS system that is being used.

There are three options:

  • iManage: set to this option when iManage integration is required
  • NetDocuments: set to this option when netDocs integration is required
  • None: (default) set to this option when no DMS integration is required
{
  "DocumentManagement": {
    "Provider": "NetDocuments",
  }
}

  • iManage
  • Net Docs

DMSKeyNames

The following DMSKEYNames can be used inf the "PropertyMappings" properties for saving into the Word document.

Document Profile information

  • profileInfo.identifier - Document number
  • profileInfo.cabinet -
  • profileInfo.version - Document version
  • profileInfo.filename - Document file name
  • profileInfo.lastModified - Date the document was last modified
  • profileInfo.createdBy - Full name of the typist

System information The other properties depend on how NetDocs is setup as will vary between different clients.

View the New Document dialog box to look at the fields and values that need to be entered:

On the left part of the screen are the list of fields these names can be used. Each has two elements .value and .description.

Example:

  • author.value eg EKT
  • author.description eg Emma Thompson
